Tutti — Politique de confidentialité

Aucune collecte de données par des tiers. Tout le contenu des publications va directement de votre navigateur vers chaque SNS.

Accueil GitHub Support

Dernière mise à jour : 2026-05-23 (v0.5.0)

1. Données que nous collectons et transmettons

Tutti n'a pas de serveur backend. Le contenu des publications (texte, images, vidéo) n'est jamais transmis à un serveur tiers. Il voyage directement de votre navigateur vers chaque réseau social que vous sélectionnez, exactement comme si vous aviez tapé et soumis vous-même.

  • Le texte, les images et la vidéo des publications sont transmis uniquement aux pages SNS que vous choisissez (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) au sein de votre navigateur.
  • Lorsque la voie d'API officielle facultative est activée pour Bluesky / Mastodon / Misskey, les publications sont envoyées directement depuis votre navigateur vers le propre serveur d'API de ce SNS en utilisant les identifiants que vous avez fournis dans les paramètres. Aucun serveur Tutti / développeur n'est impliqué.
  • Aucune donnée d'analyse, de suivi ou publicitaire n'est collectée.
  • Tutti ne lit ni ne stocke vos jetons d'authentification, cookies ou données de session des sites SNS.

Exception — rapports d'erreur manuels (consentement par clic)

Lorsque (et uniquement lorsque) vous appuyez sur le bouton « Signaler » affiché dans une boîte de dialogue d'erreur ou le panneau de diagnostic, les éléments suivants sont envoyés via un relais Cloudflare Workers vers un traqueur d'issues GitHub privé (komm64/tutti-issues, visible uniquement par le développeur de Tutti) :

  • Le message d'erreur, la version de l'extension, la chaîne User-Agent du navigateur.
  • Les 30 dernières entrées du journal interne de l'extension (le niveau de log que vous avez configuré ; INFO par défaut). Le contenu des publications, les images, la vidéo, les données de connexion SNS et les identifiants API ne sont jamais inclus.
  • Un instantané DOM expurgé de l'onglet SNS où l'erreur s'est produite, utilisé pour réparer les sélecteurs cassés. Le contenu textuel, les valeurs d'attributs, les chemins/requêtes/fragments d'URL, les attributs href/src et les balises <meta>/<link> sont supprimés. La charge utile est plafonnée à 8 Ko. Voir dom-snapshot.ts pour l'implémentation.

Rien n'est envoyé tant que vous n'appuyez pas sur le bouton de signalement. La destination est privée, visible uniquement par le développeur de Tutti.

2. Données stockées localement

Les éléments suivants sont stockés uniquement dans votre navigateur via l'API storage de Chrome. Rien n'est transmis hors de l'appareil par Tutti.

  • Paramètres (chrome.storage.sync) : URLs d'instances Mastodon / Misskey, bascule autoPost, mode d'affichage, URL d'override de sélecteurs, etc. Peut être répliqué sur vos autres appareils via Chrome Sync si vous êtes connecté à Chrome.
  • Brouillons (chrome.storage.session) : Texte de publication en cours et médias joints (images / vidéo). Effacé automatiquement à la fermeture du navigateur.
  • SNS sélectionnés (chrome.storage.local) : Les SNS que vous avez cochés comme cibles de publication. Persiste entre les sessions par commodité ; local à l'appareil uniquement.
  • Derniers noms d'utilisateur vus (chrome.storage.local) : Le nom d'utilisateur actuellement connecté sur chaque SNS, affiché dans le popup afin que vous puissiez confirmer quel compte recevra la publication. Local à l'appareil uniquement.
  • Historique des publications (chrome.storage.local) : Les 20 dernières publications : 80 premiers caractères du texte, plateformes ciblées, succès/échec par plateforme et horodatage. Local à l'appareil uniquement.
  • Identifiants API (chrome.storage.local) : Uniquement si vous avez explicitement activé la voie d'API officielle pour Bluesky / Mastodon / Misskey : votre app password Bluesky ou votre access token Mastodon / Misskey. Local à l'appareil uniquement ; jamais répliqué via Chrome Sync ; jamais transmis à un serveur du développeur. Supprimable depuis l'assistant des paramètres.
  • Cache d'override de sélecteurs (chrome.storage.local) : Données de hot-fix de sélecteurs récupérées depuis l'URL que vous avez configurée dans les paramètres (par défaut : le selectors.json public sur les GitHub Pages de Tutti).
  • Cache de limite d'upload vidéo (chrome.storage.local) : Résultat des sondages d'API pour les limites d'upload vidéo par compte (par ex. Bluesky). Valeurs numériques uniquement ; pas d'identifiants.

Tout cela peut être effacé à tout moment en désinstallant l'extension ou en vidant le stockage d'extensions de Chrome.

3. Permissions utilisées

PermissionFinalité
storageStocker les données locales ci-dessus (paramètres, brouillons, historique, plateformes sélectionnées, etc.) dans votre navigateur.
offscreenExécuter ffmpeg.wasm dans un document offscreen pour compresser / réencoder des fichiers vidéo lorsque la source dépasse la limite de taille du SNS de destination. Les données vidéo sont traitées localement uniquement ; non transmises nulle part par Tutti.
sidePanelMode d'affichage UI facultatif : afficher le formulaire de composition de Tutti dans le panneau latéral de Chrome au lieu du popup d'action, afin qu'il reste ouvert lorsque vous changez d'onglet SNS. Vous y consentez depuis la page Options ; la valeur par défaut est le popup standard.
host_permissionsInjecter des content scripts dans chaque site SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) pour remplir le formulaire de composition avec le contenu de votre publication et le soumettre en votre nom. Aucune donnée n'est lue depuis ces sites et envoyée ailleurs ; le content script n'écrit que votre propre entrée.
optional_host_permissions: https://*/*Demandé à l'exécution uniquement si vous configurez une URL d'instance personnalisée pour Mastodon ou Misskey (autre que les valeurs par défaut). Chrome affiche l'invite de permission standard à ce moment-là.

4. Relation avec les fournisseurs SNS

Tutti automatise l'interface web (ou, facultativement, publie via l'API officielle) des 11 réseaux pris en charge. Il s'agit d'une fonctionnalité non officielle ; Tutti n'est affilié ni à X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok ni YouTube. Vous restez lié aux conditions d'utilisation propres à chaque service.
