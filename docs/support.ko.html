도움이 필요하신가요?

1. 게시 실패 / 동작이 이상함

popup 내의 「진단」→「보고」 버튼을 사용하세요. 이것이 정식 채널입니다. 보고는 redact (핸들, URL, 이메일) 처리되어 비공개 issue tracker로 전달되고, 거기에서 자동으로 선택자 수정 PR이 생성되며 며칠 내에 모든 사용자에게 핫픽스로 푸시됩니다.

왜 제품 내 보고 버튼을 사용해야 하나요?

popup의 보고 흐름은 최종 작성 상태, redacted DOM 스냅샷, 선택자 감사 결과를 자동으로 첨부합니다 — 선택자 손상을 빠르게 수정하는 데 필요한 정확한 것들입니다. PII는 전송 전 여러 계층에서 제거됩니다.

2. 일반 질문 / 기능 요청

contact@komm64.com로 이메일을 보내주세요. 비공개 채널이므로 스크린샷 / 개인 핸들을 안전하게 공유할 수 있습니다.

3. 설치 / 계정 질문

Tutti는 각 소셜 네트워크의 기존 브라우저 로그인을 재사용합니다. 별도의 Tutti 계정이나 로그인 단계가 없습니다. 동일한 Chrome 프로파일에서 소셜 네트워크에 로그인되어 있으면 Tutti가 거기에서 게시할 수 있습니다.

FAQ

Q. 게시가 실패했습니다.

이는 일반적으로 소셜 네트워크가 DOM을 변경했음을 의미합니다. popup에서 진단 → 보고를 사용하세요. 선택자 핫픽스는 일반적으로 며칠 내에 selectors.json 자동 업데이트를 통해 배포됩니다 (확장 프로그램 업데이트 불필요).

Q. Tutti가 내 로그인을 기억하지 못합니다.

Tutti는 네트워크의 일반 브라우저 로그인에 의존합니다. 먼저 Chrome에서 각 네트워크에 로그인한 후 Tutti가 대신 게시할 수 있습니다. 시크릿 모드에서는 네트워크 세션이 없기 때문에 작동하지 않습니다.

Q. 네트워크별 문자 제한이 다른데 — 문제가 되나요?

아니요. 네트워크의 문자 제한을 초과하는 긴 게시물은 자동으로 연속 스레드로 분할됩니다 (X에서는 정식 답글 체인; 다른 네트워크에서는 순차 게시). 제한 내의 게시물은 그대로 전송됩니다.

Q. 이미지 / 동영상이 너무 크다고 합니다.

이미지는 Canvas를 통해 각 네트워크의 크기 제한에 맞게 자동으로 크기가 조정됩니다. 동영상은 네트워크 제한을 초과할 때 ffmpeg.wasm으로 자동 압축됩니다. 압축으로도 맞지 않는 경우 오류가 표시됩니다 — 먼저 네트워크의 네이티브 앱을 통해 파일이 업로드되는지 확인하세요.

Q. Tutti가 내 게시 콘텐츠를 보나요?

아니요. 게시 콘텐츠는 어떠한 제3자 서버도 거치지 않습니다. Tutti는 각 네트워크의 웹 작성 페이지에 직접 주입합니다. 자세한 내용은 개인정보 처리방침을 참조하세요. 유일한 예외는 제품 내 보고 버튼을 누를 때이며, 이는 redacted 오류 정보를 비공개 issue tracker로 전송합니다.

Q. 후원할 수 있나요?

아직은 아닙니다 — 부담 없는 후원 채널을 준비 중입니다. 가장 큰 지속 비용은 각 소셜 네트워크의 UI 변경을 따라잡는 것입니다; 향후의 Ko-fi 흐름은 그러한 유지보수에 자금을 지원합니다.
