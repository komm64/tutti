آخر تحديث: 2026-05-23 (v0.5.0)

1. البيانات التي نجمعها وننقلها

ليس لدى Tutti خادم واجهة خلفية. محتوى المنشور (النص والصور والفيديو) لا يُرسل أبدًا إلى أي خادم طرف ثالث. ينتقل مباشرةً من متصفحك إلى كل شبكة اجتماعية تختارها، تمامًا كما لو كنت قد كتبته وأرسلته بنفسك هناك.

  • يتم تمرير نص المنشور والصور والفيديو فقط إلى صفحات SNS التي تختارها (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) داخل متصفحك.
  • عند تمكين مسار API الرسمي الاختياري لـ Bluesky / Mastodon / Misskey، يتم إرسال المنشورات مباشرةً من متصفحك إلى خادم API الخاص بهذا الـ SNS باستخدام بيانات الاعتماد التي قدمتها في الإعدادات. لا يشارك أي خادم Tutti / المطور.
  • لا يتم جمع أي بيانات تحليلية أو تتبع أو إعلانية.
  • Tutti لا يقرأ أو يخزن رموز المصادقة أو ملفات تعريف الارتباط أو بيانات الجلسة الخاصة بك من مواقع SNS.

استثناء — تقارير الأخطاء اليدوية (موافقة لكل نقرة)

عند الضغط (وفقط عند الضغط) على زر «إبلاغ» المعروض في مربع حوار الخطأ أو لوحة التشخيص، يتم إرسال ما يلي عبر مرحل Cloudflare Workers إلى متتبع issues GitHub خاص (komm64/tutti-issues, مرئي فقط لمطور Tutti):

  • رسالة الخطأ وإصدار الإضافة وسلسلة User-Agent للمتصفح.
  • آخر 30 إدخالاً من السجل الداخلي للإضافة (مستوى السجل الذي قمت بتكوينه؛ INFO افتراضيًا). محتوى المنشورات والصور والفيديو وبيانات تسجيل الدخول إلى SNS وبيانات اعتماد API لا يتم تضمينها أبدًا.
  • لقطة DOM منقحة لعلامة تبويب SNS حيث حدث الخطأ، تُستخدم لإصلاح المحددات المعطلة. يتم تجريد محتوى النص وقيم السمات ومسارات/استعلامات/أجزاء URL وسمات href/src وعلامات <meta>/<link>. الحمولة محددة بـ 8 كيلوبايت. انظر dom-snapshot.ts للتنفيذ.

لا يتم إرسال أي شيء حتى تضغط على زر التقرير. الوجهة خاصة، مرئية فقط لمطور Tutti.

2. البيانات المخزنة محليًا

يتم تخزين ما يلي فقط داخل متصفحك عبر واجهة برمجة تطبيقات storage الخاصة بـ Chrome. لا يتم نقل أي شيء خارج الجهاز بواسطة Tutti.

  • الإعدادات (chrome.storage.sync): عناوين URL لمثيلات Mastodon / Misskey، مفتاح autoPost، وضع العرض، عنوان URL لتجاوز المحدد، إلخ. قد يتم نسخها إلى أجهزتك الأخرى عبر Chrome Sync إذا كنت قد سجلت الدخول إلى Chrome.
  • المسودات (chrome.storage.session): نص المنشور قيد التقدم والوسائط المرفقة (الصور / الفيديو). يتم مسحها تلقائيًا عند إغلاق المتصفح.
  • SNS المحددة (chrome.storage.local): SNS التي قمت بتحديدها كأهداف نشر. تستمر بين الجلسات للراحة؛ محلية فقط للجهاز.
  • أسماء المستخدمين التي شوهدت مؤخرًا (chrome.storage.local): اسم المستخدم المسجل حاليًا على كل SNS، يُعرض في النافذة المنبثقة حتى تتمكن من تأكيد الحساب الذي سيتلقى المنشور. محلية فقط للجهاز.
  • سجل المنشورات (chrome.storage.local): آخر 20 منشورًا: أول 80 حرفًا من النص، المنصات المستهدفة، النجاح/الفشل لكل منصة، والطابع الزمني. محلية فقط للجهاز.
  • بيانات اعتماد API (chrome.storage.local): فقط إذا قمت بتمكين مسار API الرسمي صراحةً لـ Bluesky / Mastodon / Misskey: Bluesky app password أو رمز الوصول الخاص بـ Mastodon / Misskey. محلية فقط للجهاز؛ لا يتم نسخها أبدًا عبر Chrome Sync؛ لا يتم نقلها أبدًا إلى أي خادم مطور. يمكن إزالتها من معالج الإعدادات.
  • ذاكرة التخزين المؤقت لتجاوز المحددات (chrome.storage.local): بيانات hot-fix للمحددات تم جلبها من عنوان URL الذي قمت بتكوينه في الإعدادات (افتراضيًا: selectors.json العام على GitHub Pages الخاص بـ Tutti).
  • ذاكرة التخزين المؤقت لحد رفع الفيديو (chrome.storage.local): نتيجة استكشاف API لحدود رفع الفيديو لكل حساب (مثل Bluesky). القيم الرقمية فقط؛ بدون بيانات اعتماد.

يمكن مسح كل هذا في أي وقت عن طريق إلغاء تثبيت الإضافة أو مسح تخزين الإضافات الخاص بـ Chrome.

3. الأذونات المستخدمة

الإذنالغرض
storageتخزين البيانات المحلية أعلاه (الإعدادات، المسودات، السجل، المنصات المحددة، إلخ) داخل متصفحك.
offscreenتشغيل ffmpeg.wasm في مستند خارج الشاشة لضغط / إعادة ترميز ملفات الفيديو عندما يتجاوز المصدر حد حجم SNS الوجهة. تتم معالجة بيانات الفيديو محليًا فقط؛ لا تُنقل إلى أي مكان بواسطة Tutti.
sidePanelوضع عرض واجهة المستخدم الاختياري: يُظهر نموذج تأليف Tutti في اللوحة الجانبية لـ Chrome بدلاً من النافذة المنبثقة للإجراء، بحيث يظل مفتوحًا أثناء التبديل بين علامات تبويب SNS. تختار من صفحة الخيارات؛ الافتراضي هو النافذة المنبثقة القياسية.
host_permissionsحقن content scripts في كل موقع SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) لملء نموذج التأليف بمحتوى منشورك وإرساله نيابةً عنك. لا تُقرأ أي بيانات من هذه المواقع وتُرسل إلى مكان آخر؛ يقوم content script فقط بكتابة إدخالك الخاص.
optional_host_permissions: https://*/*يُطلب أثناء التشغيل فقط إذا قمت بتكوين عنوان URL مخصص لمثيل Mastodon أو Misskey (غير الافتراضي). يعرض Chrome مطالبة الإذن القياسية في تلك اللحظة.

4. العلاقة مع مزودي SNS

يقوم Tutti بأتمتة واجهة المستخدم الويب (أو، اختياريًا، النشر عبر API الرسمي) لـ 11 شبكة مدعومة. هذه وظيفة غير رسمية؛ Tutti ليس مرتبطًا بـ X أو Bluesky أو Meta (Threads / Instagram) أو Mastodon gGmbH أو Misskey أو Tumblr أو Pixiv أو DeviantArt أو TikTok أو YouTube. تظل ملزمًا بشروط استخدام كل خدمة.
