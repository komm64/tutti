Tutti — 隱私權政策

無第三方資料收集。所有發布內容直接從您的瀏覽器傳送至每個 SNS。

首頁 GitHub 支援

最後更新: 2026-05-23 (v0.5.0)

1. 我們收集與傳送的資料

Tutti 沒有後端伺服器。 發布內容 (文字、圖片、影片) 絕不會傳送至任何第三方伺服器。它直接從您的瀏覽器送至您選擇的每個社群網路,如同您親自在該處輸入並送出一樣。

  • 發布的文字、圖片與影片僅在您的瀏覽器中傳遞給您選擇的 SNS 頁面 (X、Bluesky、Threads、Mastodon、Misskey、Tumblr、Pixiv、DeviantArt、Instagram、TikTok、YouTube)。
  • 若您為 Bluesky / Mastodon / Misskey 啟用了選用的官方 API 路徑,發布內容將使用您在設定中提供的憑證,直接從瀏覽器送至 該 SNS 自身的 API 伺服器。Tutti / 開發者伺服器不參與其中。
  • 不收集任何分析、追蹤或廣告資料。
  • Tutti 不會 讀取或儲存您在 SNS 站點上的驗證 token、cookie 或工作階段資料。

例外 — 手動錯誤回報 (每次點擊即同意)

當 (且僅當) 您按下錯誤對話框或診斷面板中顯示的 「回報」按鈕 時,以下內容會透過 Cloudflare Workers 中繼傳送至 私有的 GitHub issue tracker (komm64/tutti-issues,僅 Tutti 開發者可見):

  • 錯誤訊息、擴充功能版本、瀏覽器 User-Agent 字串。
  • 擴充功能內部記錄的最後 30 筆 (依您設定的記錄層級;預設 INFO)。發布內容、圖片、影片、SNS 登入資料和 API 憑證 絕不 包含在內。
  • 發生錯誤的 SNS 分頁的 redacted DOM 快照,用以修復失效的選擇器。文字內容、屬性值、URL 路徑/查詢/片段、href/src 屬性以及 <meta>/<link> 標籤皆已剝除。負載上限為 8 KB。實作請參見 dom-snapshot.ts。

在您按下回報按鈕前不會傳送任何資料。接收方為私有,僅 Tutti 開發者可見。

2. 本機儲存的資料

以下資料僅透過 Chrome 的 storage API 儲存於您的瀏覽器內。Tutti 不會將其傳送至裝置外。

  • 設定 (chrome.storage.sync): Mastodon / Misskey 執行個體 URL、autoPost 開關、顯示方式、selector override URL 等。若您已登入 Chrome,可能透過 Chrome Sync 同步至您的其他裝置。
  • 草稿 (chrome.storage.session): 正在編輯的發布文字及附加媒體 (圖片 / 影片)。瀏覽器結束時自動清除。
  • 已選 SNS (chrome.storage.local): 您勾選為發布目標的 SNS。為方便起見會跨工作階段保留;僅限本機。
  • 最後所見使用者名稱 (chrome.storage.local): 各 SNS 目前登入的使用者名稱,顯示在 popup 中以便您確認哪個帳號將接收發布。僅限本機。
  • 發布記錄 (chrome.storage.local): 最近 20 筆發布: 文字的前 80 個字元、目標平台、每個平台的成功/失敗狀態、時間戳記。僅限本機。
  • API 憑證 (chrome.storage.local): 僅當您為 Bluesky / Mastodon / Misskey 明確啟用官方 API 路徑時儲存: 您的 Bluesky app password 或 Mastodon / Misskey access token。僅限本機;絕不透過 Chrome Sync 同步;絕不傳送至任何開發者伺服器。 可從設定精靈中移除。
  • 選擇器覆寫快取 (chrome.storage.local): 從您在設定中設定的 URL 取得的選擇器熱修復資料 (預設: Tutti 於 GitHub Pages 上公開的 selectors.json)。
  • 影片上傳上限快取 (chrome.storage.local): 針對每個帳號影片上傳上限 (例如 Bluesky) 的 API 探測結果。僅數值;不含憑證。

解除安裝擴充功能或清除 Chrome 的擴充功能儲存即可隨時清除上述所有資料。

3. 使用的權限

權限用途
storage在瀏覽器內儲存上述本機資料 (設定、草稿、記錄、選擇的平台等)。
offscreen在 offscreen 文件中執行 ffmpeg.wasm,當來源超過目標 SNS 大小上限時壓縮 / 重新編碼影片檔案。影片資料僅在本機處理;Tutti 不會將其傳送至任何位置。
sidePanel選用的 UI 顯示方式: 在 Chrome 的側邊面板顯示 Tutti 的發布表單,而非操作 popup,讓您在 SNS 分頁間切換時能保持開啟。可於選項頁面啟用;預設為標準 popup。
host_permissions將 content script 注入各 SNS 站點 (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com),以代您填寫發布表單並送出。不會從這些站點讀取資料並送至其他位置;content script 僅寫入您自身的輸入。
optional_host_permissions: https://*/*僅當您設定自訂 Mastodon 或 Misskey 執行個體 URL (預設值以外) 時才會在執行階段請求。屆時 Chrome 會顯示標準權限提示。

4. 與 SNS 提供者的關係

Tutti 自動化 (或選擇性地透過官方 API) 操作 11 個支援網路的 Web UI。這是非官方功能;Tutti 與 X、Bluesky、Meta (Threads / Instagram)、Mastodon gGmbH、Misskey、Tumblr、Pixiv、DeviantArt、TikTok 或 YouTube 均無關係。您仍須遵守每項服務自身的使用條款。
