Tutti — Informativa sulla privacy

Nessuna raccolta di dati da parte di terzi. Tutto il contenuto dei post va direttamente dal tuo browser a ciascun SNS.

Ultimo aggiornamento: 2026-05-23 (v0.5.0)

1. Dati che raccogliamo e trasmettiamo

Tutti non ha un server backend. Il contenuto dei post (testo, immagini, video) non viene mai trasmesso ad alcun server di terze parti. Viaggia direttamente dal tuo browser a ciascun social network selezionato, esattamente come se lo avessi digitato e inviato tu stesso.

  • Il testo, le immagini e i video dei post vengono passati solo alle pagine SNS che scegli (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) all'interno del tuo browser.
  • Quando il percorso API ufficiale facoltativo è abilitato per Bluesky / Mastodon / Misskey, i post vengono inviati direttamente dal tuo browser al server API proprio di quel SNS utilizzando le credenziali da te fornite nelle Impostazioni. Nessun server Tutti / dello sviluppatore è coinvolto.
  • Non vengono raccolti dati di analisi, tracciamento o pubblicitari.
  • Tutti non legge né memorizza i tuoi token di autenticazione, cookie o dati di sessione dei siti SNS.

Eccezione — segnalazioni manuali di errore (consenso per clic)

Quando (e solo quando) premi il pulsante “Segnala” mostrato in una finestra di errore o nel pannello di diagnosi, il seguente viene inviato tramite un relay Cloudflare Workers a un tracker di issue GitHub privato (komm64/tutti-issues, visibile solo allo sviluppatore di Tutti):

  • Il messaggio di errore, la versione dell'estensione, la stringa User-Agent del browser.
  • Le ultime 30 voci del log interno dell'estensione (il livello di log che hai configurato; INFO per impostazione predefinita). Il contenuto dei post, le immagini, i video, i dati di accesso ai SNS e le credenziali API non sono mai inclusi.
  • Uno snapshot DOM redatto della scheda SNS in cui si è verificato l'errore, utilizzato per riparare i selettori rotti. Il contenuto testuale, i valori degli attributi, i percorsi/query/frammenti URL, gli attributi href/src e i tag <meta>/<link> vengono rimossi. Il payload è limitato a 8 KB. Vedere dom-snapshot.ts per l'implementazione.

Nulla viene inviato finché non premi il pulsante di segnalazione. La destinazione è privata, visibile solo allo sviluppatore di Tutti.

2. Dati memorizzati localmente

Quanto segue è memorizzato solo all'interno del tuo browser tramite l'API storage di Chrome. Nulla viene trasmesso fuori dal dispositivo da Tutti.

  • Impostazioni (chrome.storage.sync): URL di istanze Mastodon / Misskey, toggle autoPost, modalità di visualizzazione, URL di override dei selettori, ecc. Può essere replicato sui tuoi altri dispositivi tramite Chrome Sync se hai effettuato l'accesso a Chrome.
  • Bozze (chrome.storage.session): Testo del post in corso e media allegati (immagini / video). Cancellato automaticamente alla chiusura del browser.
  • SNS selezionati (chrome.storage.local): I SNS che hai contrassegnato come destinazioni di pubblicazione. Persiste tra le sessioni per comodità; solo locale al dispositivo.
  • Ultimi nomi utente visti (chrome.storage.local): Il nome utente attualmente connesso su ciascun SNS, visualizzato nel popup per consentirti di confermare quale account riceverà il post. Solo locale al dispositivo.
  • Cronologia post (chrome.storage.local): Gli ultimi 20 post: primi 80 caratteri di testo, piattaforme di destinazione, successo/fallimento per piattaforma e timestamp. Solo locale al dispositivo.
  • Credenziali API (chrome.storage.local): Solo se hai abilitato esplicitamente il percorso API ufficiale per Bluesky / Mastodon / Misskey: la tua app password Bluesky o l'access token Mastodon / Misskey. Solo locale al dispositivo; mai replicato tramite Chrome Sync; mai trasmesso ad alcun server dello sviluppatore. Rimovibile dalla procedura guidata delle Impostazioni.
  • Cache di override dei selettori (chrome.storage.local): Dati di hot-fix dei selettori recuperati dall'URL che hai configurato in Impostazioni (predefinito: il selectors.json pubblico sulle GitHub Pages di Tutti).
  • Cache del limite di upload video (chrome.storage.local): Risultato dei sondaggi API per i limiti di upload video per account (ad es. Bluesky). Solo valori numerici; nessuna credenziale.

Tutto questo può essere cancellato in qualsiasi momento disinstallando l'estensione o cancellando lo storage delle estensioni di Chrome.

3. Autorizzazioni utilizzate

AutorizzazioneScopo
storageMemorizzare i dati locali sopra (impostazioni, bozze, cronologia, piattaforme selezionate, ecc.) all'interno del tuo browser.
offscreenEseguire ffmpeg.wasm in un documento offscreen per comprimere / ricodificare file video quando la sorgente supera il limite di dimensione del SNS di destinazione. I dati video vengono elaborati solo localmente; non trasmessi da nessuna parte da Tutti.
sidePanelModalità di visualizzazione UI opzionale: mostra il modulo di composizione di Tutti nel pannello laterale di Chrome anziché nel popup di azione, in modo che rimanga aperto mentre passi da una scheda SNS all'altra. Opt-in dalla pagina Opzioni; il valore predefinito è il popup standard.
host_permissionsIniettare content script in ciascun sito SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) per compilare il modulo di composizione con il contenuto del post e inviarlo per tuo conto. Nessun dato viene letto da questi siti e inviato altrove; il content script scrive solo il tuo input.
optional_host_permissions: https://*/*Richiesto a runtime solo se configuri un URL di istanza Mastodon o Misskey personalizzato (diverso dai predefiniti). In quel momento Chrome mostra il prompt di autorizzazione standard.

4. Rapporto con i provider SNS

Tutti automatizza l'interfaccia web (o, facoltativamente, pubblica tramite l'API ufficiale) degli 11 network supportati. Questa è funzionalità non ufficiale; Tutti non è affiliato a X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok o YouTube. Rimani soggetto ai termini di utilizzo di ciascun servizio.
