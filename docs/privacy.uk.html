Ніякого збору даних третіми сторонами. Весь вміст дописів йде прямо з вашого браузера в кожну SNS.

Головна GitHub Підтримка

Останнє оновлення: 2026-05-23 (v0.5.0)

1. Дані, які ми збираємо та передаємо

У Tutti немає бекенд-сервера. Вміст дописів (текст, зображення, відео) ніколи не передається жодному сторонньому серверу. Він мандрує прямо з вашого браузера в кожну вибрану вами соцмережу, точно так, ніби ви самі набрали і надіслали його там.

  • Текст, зображення та відео дописів передаються лише на вибрані вами сторінки SNS (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) у вашому браузері.
  • Коли увімкнено опціональний шлях через офіційний API для Bluesky / Mastodon / Misskey, дописи надсилаються прямо з вашого браузера на власний API-сервер цього SNS з використанням облікових даних, які ви вказали в Налаштуваннях. Жоден сервер Tutti / розробника не задіяний.
  • Жодні аналітичні, трекінгові чи рекламні дані не збираються.
  • Tutti не читає і не зберігає ваші токени автентифікації, кукі або дані сесії з сайтів SNS.

Виняток — ручні звіти про помилки (згода на кожен клік)

Коли (і лише коли) ви натискаєте кнопку «Повідомити», показану в діалозі помилки або панелі діагностики, через релей Cloudflare Workers у приватний трекер завдань GitHub (komm64/tutti-issues, видимий лише розробнику Tutti) надсилається наступне:

  • Повідомлення про помилку, версія розширення, рядок User-Agent браузера.
  • Останні 30 записів внутрішнього журналу розширення (налаштований вами рівень логу; за замовчуванням INFO). Вміст дописів, зображення, відео, дані входу в SNS та облікові дані API ніколи не включаються.
  • Відредагований DOM-знімок вкладки SNS, де сталася помилка, використовується для лагодження зламаних селекторів. Текстовий вміст, значення атрибутів, шляхи/запити/фрагменти URL, атрибути href/src та теги <meta>/<link> видаляються. Розмір корисного навантаження обмежений 8 КБ. Реалізацію див. у dom-snapshot.ts.

Нічого не надсилається, поки ви не натиснете кнопку звіту. Призначення приватне, видиме лише розробнику Tutti.

2. Локально збережені дані

Наступне зберігається лише всередині вашого браузера через API storage Chrome. Tutti не передає нічого за межі пристрою.

  • Налаштування (chrome.storage.sync): URL-адреси інстансів Mastodon / Misskey, перемикач autoPost, режим відображення, URL перевизначення селекторів тощо. Може реплікуватися на ваші інші пристрої через Chrome Sync, якщо ви увійшли в Chrome.
  • Чернетки (chrome.storage.session): Текст допису у процесі та прикріплені медіа (зображення / відео). Автоматично очищається при закритті браузера.
  • Вибрані SNS (chrome.storage.local): SNS, які ви позначили як цілі публікації. Зберігається між сесіями для зручності; лише локально на пристрої.
  • Останні побачені імена користувачів (chrome.storage.local): Поточне ім'я користувача, що увійшов у кожну SNS, відображається в popup, щоб ви могли підтвердити, який обліковий запис отримає допис. Лише локально на пристрої.
  • Історія дописів (chrome.storage.local): Останні 20 дописів: перші 80 символів тексту, цільові платформи, успіх/невдача за платформою та часова мітка. Лише локально на пристрої.
  • Облікові дані API (chrome.storage.local): Лише якщо ви явно увімкнули шлях через офіційний API для Bluesky / Mastodon / Misskey: ваш Bluesky app password або токен доступу Mastodon / Misskey. Лише локально на пристрої; ніколи не реплікується через Chrome Sync; ніколи не передається на жоден сервер розробника. Видаляється з майстра Налаштувань.
  • Кеш перевизначення селекторів (chrome.storage.local): Дані гарячого виправлення селекторів, отримані з URL, який ви налаштували в Налаштуваннях (за замовчуванням: публічний selectors.json на GitHub Pages Tutti).
  • Кеш ліміту завантаження відео (chrome.storage.local): Результат API-зондів для лімітів завантаження відео по обліковому запису (наприклад, Bluesky). Лише числові значення; без облікових даних.

Усе це можна стерти в будь-який час, видаливши розширення або очистивши сховище розширень Chrome.

3. Використовувані дозволи

ДозвілПризначення
storageЗберігати вищезгадані локальні дані (налаштування, чернетки, історія, вибрані платформи тощо) всередині вашого браузера.
offscreenЗапускати ffmpeg.wasm в офскрин-документі для стискання / перекодування відеофайлів, коли джерело перевищує обмеження розміру цільового SNS. Відеодані обробляються лише локально; Tutti не передає їх нікуди.
sidePanelОпціональний режим відображення UI: показує форму створення допису Tutti в бічній панелі Chrome замість popup дії, щоб вона залишалася відкритою, коли ви перемикаєтеся між вкладками SNS. Ви вмикаєте це на сторінці Параметрів; за замовчуванням використовується звичайний popup.
host_permissionsВпроваджувати контент-скрипти в кожен сайт SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com), щоб заповнити форму створення допису вмістом вашого допису та надіслати його від вашого імені. Жодних даних не зчитується з цих сайтів і не надсилається кудись ще; контент-скрипт записує лише ваш власний ввід.
optional_host_permissions: https://*/*Запитується під час виконання, лише якщо ви налаштовуєте користувацький URL інстансу Mastodon або Misskey (відмінний від значень за замовчуванням). Chrome показує стандартний запит дозволу в цей момент.

4. Стосунки з провайдерами SNS

Tutti автоматизує веб-інтерфейс (або, опціонально, публікує через офіційний API) 11 підтримуваних мереж. Це неофіційна функціональність; Tutti не пов'язаний з X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok або YouTube. Ви продовжуєте дотримуватися власних умов використання кожного сервісу.
