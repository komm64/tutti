Tutti — Política de privacidade

Sem coleta de dados por terceiros. Todo o conteúdo de postagens vai diretamente do seu navegador para cada SNS.

Início GitHub Suporte

Última atualização: 2026-05-23 (v0.5.0)

1. Dados que coletamos e transmitimos

O Tutti não tem servidor backend. O conteúdo das postagens (texto, imagens, vídeo) nunca é transmitido para nenhum servidor de terceiros. Ele viaja diretamente do seu navegador para cada rede social que você selecionar, exatamente como se você tivesse digitado e enviado lá mesmo.

  • O texto, as imagens e os vídeos das postagens são passados apenas para as páginas dos SNS que você escolher (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) dentro do seu navegador.
  • Quando o caminho opcional de API oficial está habilitado para Bluesky / Mastodon / Misskey, as postagens são enviadas diretamente do seu navegador para o próprio servidor de API daquele SNS usando as credenciais que você forneceu nas Configurações. Nenhum servidor do Tutti / desenvolvedor está envolvido.
  • Nenhum dado analítico, de rastreamento ou publicitário é coletado.
  • O Tutti não lê nem armazena seus tokens de autenticação, cookies ou dados de sessão dos sites SNS.

Exceção — relatórios manuais de erro (consentimento por clique)

Quando (e somente quando) você pressiona o botão “Reportar” mostrado em uma caixa de diálogo de erro ou no painel de diagnóstico, o seguinte é enviado através de um relay do Cloudflare Workers para um rastreador privado de issues do GitHub (komm64/tutti-issues, visível apenas para o desenvolvedor do Tutti):

  • A mensagem de erro, a versão da extensão, a string User-Agent do navegador.
  • As últimas 30 entradas do log interno da extensão (o nível de log que você configurou; INFO por padrão). Conteúdo de postagens, imagens, vídeo, dados de login em SNS e credenciais de API nunca são incluídos.
  • Um snapshot DOM redigido da aba SNS onde o erro ocorreu, usado para reparar seletores quebrados. O conteúdo de texto, os valores de atributos, os caminhos/consultas/fragmentos de URL, os atributos href/src e as tags <meta>/<link> são removidos. O payload é limitado a 8 KB. Veja dom-snapshot.ts para a implementação.

Nada é enviado até você pressionar o botão de relatório. O destino é privado, visível apenas para o desenvolvedor do Tutti.

2. Dados armazenados localmente

O seguinte é armazenado apenas dentro do seu navegador através da API storage do Chrome. Nada é transmitido para fora do dispositivo pelo Tutti.

  • Configurações (chrome.storage.sync): URLs de instâncias Mastodon / Misskey, toggle autoPost, modo de exibição, URL de override de seletores, etc. Pode ser replicado para seus outros dispositivos via Chrome Sync se você estiver conectado ao Chrome.
  • Rascunhos (chrome.storage.session): Texto de postagem em andamento e mídia anexada (imagens / vídeo). Limpo automaticamente quando o navegador é fechado.
  • SNS selecionados (chrome.storage.local): Os SNS que você marcou como destinos de postagem. Persiste entre sessões por conveniência; apenas local no dispositivo.
  • Últimos nomes de usuário vistos (chrome.storage.local): O nome de usuário atualmente conectado em cada SNS, exibido no popup para você confirmar qual conta receberá a postagem. Apenas local no dispositivo.
  • Histórico de postagens (chrome.storage.local): As últimas 20 postagens: primeiros 80 caracteres do texto, plataformas-alvo, sucesso/falha por plataforma e timestamp. Apenas local no dispositivo.
  • Credenciais de API (chrome.storage.local): Apenas se você habilitou explicitamente o caminho de API oficial para Bluesky / Mastodon / Misskey: seu app password do Bluesky ou access token do Mastodon / Misskey. Apenas local no dispositivo; nunca replicado via Chrome Sync; nunca transmitido para nenhum servidor do desenvolvedor. Removível pelo assistente de Configurações.
  • Cache de override de seletores (chrome.storage.local): Dados de hot-fix de seletores obtidos da URL que você configurou em Configurações (padrão: o selectors.json público nas GitHub Pages do Tutti).
  • Cache de limite de upload de vídeo (chrome.storage.local): Resultado de sondagens de API para limites de upload de vídeo por conta (p. ex., Bluesky). Apenas valores numéricos; sem credenciais.

Tudo isso pode ser apagado a qualquer momento desinstalando a extensão ou limpando o armazenamento de extensões do Chrome.

3. Permissões usadas

PermissãoPropósito
storageArmazenar os dados locais acima (configurações, rascunhos, histórico, plataformas selecionadas, etc.) dentro do seu navegador.
offscreenExecutar ffmpeg.wasm em um documento offscreen para comprimir / recodificar arquivos de vídeo quando a origem exceder o limite de tamanho do SNS de destino. Dados de vídeo são processados apenas localmente; não transmitidos a lugar nenhum pelo Tutti.
sidePanelModo de exibição de UI opcional: mostra o formulário de redação do Tutti no painel lateral do Chrome em vez do popup de ação, para que permaneça aberto enquanto você alterna entre abas de SNS. Você opta pela página de Opções; o padrão é o popup padrão.
host_permissionsInjetar content scripts em cada site SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) para preencher o formulário de redação com o conteúdo da sua postagem e enviá-lo em seu nome. Nenhum dado é lido desses sites e enviado para outro lugar; o content script apenas escreve sua própria entrada.
optional_host_permissions: https://*/*Solicitado em tempo de execução apenas se você configurar uma URL de instância personalizada de Mastodon ou Misskey (diferente das padrões). O Chrome mostra o prompt de permissão padrão nesse momento.

4. Relação com os provedores de SNS

O Tutti automatiza a UI web das (ou, opcionalmente, posta via a API oficial das) 11 redes suportadas. Esta é uma funcionalidade não oficial; o Tutti não é afiliado a X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok ou YouTube. Você permanece sujeito aos termos de uso de cada serviço.
