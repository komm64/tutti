Tutti — 隐私政策

无第三方数据收集。所有发布内容直接从您的浏览器发送至每个 SNS。

首页 GitHub 支持

最后更新: 2026-05-23 (v0.5.0)

1. 我们收集和传输的数据

Tutti 没有后端服务器。 发布内容 (文本、图片、视频) 绝不传输至任何第三方服务器。它直接从您的浏览器发送至您选择的每个社交网络,如同您亲自在该处输入并提交一般。

  • 发布的文本、图片和视频仅在您的浏览器中传递给您选择的 SNS 页面 (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube)。
  • 若您为 Bluesky / Mastodon / Misskey 启用了可选的官方 API 路径,发布内容将使用您在设置中提供的凭据,直接从浏览器发送至 该 SNS 自身的 API 服务器。Tutti / 开发者服务器不参与其中。
  • 不收集任何分析、追踪或广告数据。
  • Tutti 不会 读取或存储您在 SNS 站点上的认证 token、cookie 或会话数据。

例外 — 手动错误报告 (每次点击即时同意)

当 (且仅当) 您按下错误对话框或诊断面板中显示的 「报告」按钮 时,以下内容会通过 Cloudflare Workers 中继发送至 私有的 GitHub issue tracker (komm64/tutti-issues,仅 Tutti 开发者可见):

  • 错误消息、扩展版本、浏览器 User-Agent 字符串。
  • 扩展内部日志的最后 30 条 (按您配置的日志级别;默认 INFO)。发布内容、图片、视频、SNS 登录数据和 API 凭据 绝不 包含在内。
  • 发生错误的 SNS 标签页的 redacted DOM 快照,用于修复失效的选择器。文本内容、属性值、URL 路径/查询/片段、href/src 属性以及 <meta>/<link> 标签均被剥离。负载上限为 8 KB。实现请参见 dom-snapshot.ts。

您按下报告按钮前不会发送任何数据。接收方为私有,仅 Tutti 开发者可见。

2. 本地存储的数据

以下数据仅通过 Chrome 的 storage API 存储于您的浏览器内。Tutti 不会将其传输至设备外。

  • 设置 (chrome.storage.sync): Mastodon / Misskey 实例 URL、autoPost 开关、显示方式、selector override URL 等。若您已登录 Chrome,可能通过 Chrome Sync 复制到您的其他设备。
  • 草稿 (chrome.storage.session): 正在编辑的发布文本及附加媒体 (图片 / 视频)。浏览器退出时自动清除。
  • 选中的 SNS (chrome.storage.local): 您勾选为发布目标的 SNS。为方便起见在多次会话间保留;仅设备本地。
  • 最后所见用户名 (chrome.storage.local): 各 SNS 当前登录的用户名,显示在 popup 中以便您确认哪个账号将接收发布。仅设备本地。
  • 发布历史 (chrome.storage.local): 最近 20 次发布: 文本的前 80 个字符、目标平台、每个平台的成功/失败状态、时间戳。仅设备本地。
  • API 凭据 (chrome.storage.local): 仅当您为 Bluesky / Mastodon / Misskey 明确启用官方 API 路径时存储: 您的 Bluesky app password 或 Mastodon / Misskey access token。仅设备本地;绝不通过 Chrome Sync 复制;绝不传输至任何开发者服务器。 可从设置向导中删除。
  • 选择器覆盖缓存 (chrome.storage.local): 从您在设置中配置的 URL 获取的选择器热修复数据 (默认: Tutti 在 GitHub Pages 上的公共 selectors.json)。
  • 视频上传上限缓存 (chrome.storage.local): 针对每账号视频上传上限 (例如 Bluesky) 的 API 探测结果。仅数值,不含凭据。

卸载扩展或清除 Chrome 的扩展存储即可随时擦除以上所有数据。

3. 所用权限

权限用途
storage在浏览器内存储上述本地数据 (设置、草稿、历史、选中的平台等)。
offscreen在 offscreen 文档中运行 ffmpeg.wasm,当源文件超过目标 SNS 大小上限时压缩 / 重新编码视频文件。视频数据仅本地处理;Tutti 不会将其传输至任何位置。
sidePanel可选的 UI 显示方式: 在 Chrome 的侧边栏中显示 Tutti 的发布表单,而非操作 popup,这样在切换 SNS 标签时它能保持打开。您可在选项页中启用;默认为标准 popup。
host_permissions将 content script 注入各 SNS 站点 (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com),以代您填写发布表单内容并提交。不会从这些站点读取数据并发送至其他位置;content script 仅写入您自己的输入。
optional_host_permissions: https://*/*仅当您配置自定义 Mastodon 或 Misskey 实例 URL (默认值之外) 时才在运行时请求。届时 Chrome 会显示标准权限提示。

4. 与 SNS 提供商的关系

Tutti 自动化 (或通过官方 API) 操作 11 个支持的网络的 Web UI。这是非官方功能;Tutti 与 X、Bluesky、Meta (Threads / Instagram)、Mastodon gGmbH、Misskey、Tumblr、Pixiv、DeviantArt、TikTok 或 YouTube 均无附属关系。您仍需遵守每项服务自身的使用条款。
