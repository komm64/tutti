需要帮助?

1. 发布失败 / 有些行为异常

请使用 popup 中的「诊断」→「报告」按钮。这是正规渠道。报告经过 redact (用户名、URL、邮箱) 后转发至我们的私有 issue tracker,在那里会自动创建 selector 修复 PR,数天内以热修复形式推送至全体用户。

为什么使用产品内的报告按钮?

popup 的报告流程会自动附上最终发布状态、redacted DOM 快照及选择器审计结果 — 这正是快速修复 selector 失效所需的内容。个人可识别信息在发送前经过多层 redact。

2. 一般问题 / 功能请求

请发邮件至 contact@komm64.com。这是私有渠道,可以安全分享截图 / 个人用户名。

3. 安装 / 账号问题

Tutti 重用您各个社交网络的现有浏览器登录。无需单独的 Tutti 账号或登录步骤。只要您在同一个 Chrome 配置文件中已登录某个社交网络,Tutti 就能从那里发布。

FAQ

Q. 我的发布失败了。

通常意味着某个社交网络更改了其 DOM。请使用 popup 中的 诊断 → 报告。选择器热修复通常会在数天内通过 selectors.json 自动更新发布 (无需更新扩展)。

Q. Tutti 记不住我的登录。

Tutti 依赖网络的常规浏览器登录。请先在 Chrome 中登录每个网络,然后 Tutti 才能代您发布。隐身模式不可用,因为那里没有该网络的会话。

Q. 每个网络的字符上限不同 — 这有问题吗?

没有问题。超过某网络字符上限的长文会自动拆分为连续推文 (在 X 上是真正的回复链;其他网络是顺序发布)。在上限以内的发布则原样发送。

Q. 提示我的图片 / 视频过大。

图片通过 Canvas 自动调整大小以适应各网络的大小上限。视频在超过网络上限时通过 ffmpeg.wasm 自动压缩。若压缩后仍无法适应,您将看到错误 — 请先通过该网络的原生应用验证文件可上传。

Q. Tutti 能看到我的发布内容吗?

不能。发布内容绝不经过任何第三方服务器。Tutti 直接注入各网络的 Web 发布页面。详情请参阅我们的隐私政策。唯一例外是您按下产品内报告按钮时,这会将 redacted 错误信息发送至我们的私有 issue tracker。

Q. 我可以捐赠吗?

暂时还不行 — 我们正在准备一个低压力的支持渠道。最大的持续成本是跟进每个社交网络的 UI 变更;未来的 Ko-fi 流程将资助这种维护。
