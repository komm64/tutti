Tutti — นโยบายความเป็นส่วนตัว

ไม่มีการเก็บข้อมูลโดยบุคคลที่สาม เนื้อหาโพสต์ทั้งหมดส่งตรงจากเบราว์เซอร์ของคุณไปยัง SNS แต่ละแห่ง

หน้าแรก GitHub การสนับสนุน

อัปเดตล่าสุด: 2026-05-23 (v0.5.0)

1. ข้อมูลที่เราเก็บและส่งต่อ

Tutti ไม่มีเซิร์ฟเวอร์แบ็กเอนด์ เนื้อหาโพสต์ (ข้อความ, ภาพ, วิดีโอ) ไม่ถูกส่งไปยังเซิร์ฟเวอร์บุคคลที่สามใด ๆ ทั้งสิ้น มันเดินทางตรงจากเบราว์เซอร์ของคุณไปยังเครือข่ายสังคมแต่ละแห่งที่คุณเลือก เหมือนกับว่าคุณพิมพ์และส่งด้วยตัวเองที่นั่น

  • ข้อความโพสต์, ภาพ, และวิดีโอจะถูกส่งไปยังหน้า SNS ที่คุณเลือกเท่านั้น (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) ภายในเบราว์เซอร์ของคุณ
  • เมื่อเส้นทาง API อย่างเป็นทางการแบบเสริมถูกเปิดใช้งานสำหรับ Bluesky / Mastodon / Misskey โพสต์จะถูกส่งตรงจากเบราว์เซอร์ของคุณไปยัง เซิร์ฟเวอร์ API ของ SNS นั้นเอง โดยใช้ข้อมูลรับรองที่คุณให้ไว้ในการตั้งค่า ไม่มีเซิร์ฟเวอร์ Tutti / ผู้พัฒนาเกี่ยวข้อง
  • ไม่มีการเก็บข้อมูลการวิเคราะห์, การติดตาม, หรือโฆษณา
  • Tutti ไม่ อ่านหรือเก็บโทเค็นการรับรอง, คุกกี้, หรือข้อมูลเซสชันจากเว็บไซต์ SNS

ข้อยกเว้น — รายงานข้อผิดพลาดด้วยตนเอง (ยินยอมต่อการคลิก)

เมื่อ (และเฉพาะเมื่อ) คุณกด ปุ่ม «รายงาน» ที่แสดงในกล่องโต้ตอบข้อผิดพลาดหรือแผงวินิจฉัย ข้อมูลต่อไปนี้จะถูกส่งผ่าน Cloudflare Workers relay ไปยัง ตัวติดตามประเด็น GitHub แบบส่วนตัว (komm64/tutti-issues, มองเห็นได้เฉพาะผู้พัฒนา Tutti เท่านั้น):

  • ข้อความข้อผิดพลาด, เวอร์ชันส่วนขยาย, สตริง User-Agent ของเบราว์เซอร์
  • 30 รายการล่าสุดของบันทึกภายในของส่วนขยาย (ระดับบันทึกที่คุณตั้งค่า; INFO ตามค่าเริ่มต้น) เนื้อหาโพสต์, ภาพ, วิดีโอ, ข้อมูลล็อกอิน SNS, และข้อมูลรับรอง API ไม่ ถูกรวมไว้
  • ภาพรวม DOM ที่ผ่านการแก้ไขข้อมูลของแท็บ SNS ที่เกิดข้อผิดพลาด ใช้สำหรับซ่อมแซมตัวเลือกที่เสีย เนื้อหาข้อความ, ค่าแอตทริบิวต์, เส้นทาง/คำสั่ง/ส่วนของ URL, แอตทริบิวต์ href/src และแท็ก <meta>/<link> ถูกตัดออก ขนาด payload ถูกจำกัดที่ 8 KB ดูการนำไปใช้ที่ dom-snapshot.ts

ไม่มีอะไรถูกส่งจนกว่าคุณจะกดปุ่มรายงาน ปลายทางเป็นส่วนตัว มองเห็นได้เฉพาะผู้พัฒนา Tutti เท่านั้น

2. ข้อมูลที่จัดเก็บไว้ในเครื่อง

ข้อมูลต่อไปนี้ถูกจัดเก็บไว้ภายในเบราว์เซอร์ของคุณเท่านั้นผ่าน Chrome storage API ไม่มีสิ่งใดถูกส่งออกจากอุปกรณ์โดย Tutti

  • การตั้งค่า (chrome.storage.sync): URL อินสแตนซ์ Mastodon / Misskey, สวิตช์ autoPost, โหมดการแสดงผล, URL แทนที่ตัวเลือก ฯลฯ อาจถูกจำลองไปยังอุปกรณ์อื่น ๆ ของคุณผ่าน Chrome Sync หากคุณลงชื่อเข้าใช้ Chrome
  • ฉบับร่าง (chrome.storage.session): ข้อความโพสต์ที่กำลังดำเนินการและสื่อที่แนบ (ภาพ / วิดีโอ) ล้างอัตโนมัติเมื่อปิดเบราว์เซอร์
  • SNS ที่เลือก (chrome.storage.local): SNS ที่คุณเลือกเป็นเป้าหมายการโพสต์ คงอยู่ระหว่างเซสชันเพื่อความสะดวก; เฉพาะในอุปกรณ์เท่านั้น
  • ชื่อผู้ใช้ที่เห็นล่าสุด (chrome.storage.local): ชื่อผู้ใช้ที่เข้าสู่ระบบปัจจุบันในแต่ละ SNS แสดงใน popup เพื่อให้คุณยืนยันว่าบัญชีใดจะรับโพสต์ เฉพาะในอุปกรณ์เท่านั้น
  • ประวัติโพสต์ (chrome.storage.local): 20 โพสต์ล่าสุด: 80 อักขระแรกของข้อความ, แพลตฟอร์มเป้าหมาย, สำเร็จ/ล้มเหลวตามแพลตฟอร์ม, และตราเวลา เฉพาะในอุปกรณ์เท่านั้น
  • ข้อมูลรับรอง API (chrome.storage.local): เฉพาะเมื่อคุณเปิดใช้งานเส้นทาง API อย่างเป็นทางการสำหรับ Bluesky / Mastodon / Misskey อย่างชัดเจน: Bluesky app password ของคุณ หรือโทเค็นการเข้าถึง Mastodon / Misskey เฉพาะในอุปกรณ์เท่านั้น; ไม่เคยจำลองผ่าน Chrome Sync; ไม่เคยส่งไปยังเซิร์ฟเวอร์ผู้พัฒนาใด ๆ สามารถลบได้จากตัวช่วยตั้งค่า
  • แคชการแทนที่ตัวเลือก (chrome.storage.local): ข้อมูล hot-fix ตัวเลือกที่ดึงจาก URL ที่คุณกำหนดในการตั้งค่า (ค่าเริ่มต้น: selectors.json สาธารณะบน GitHub Pages ของ Tutti)
  • แคชขีดจำกัดการอัปโหลดวิดีโอ (chrome.storage.local): ผลลัพธ์ของการตรวจสอบ API สำหรับขีดจำกัดการอัปโหลดวิดีโอต่อบัญชี (เช่น Bluesky) เฉพาะค่าตัวเลข; ไม่มีข้อมูลรับรอง

สิ่งเหล่านี้ทั้งหมดสามารถลบได้ตลอดเวลาโดยการถอนการติดตั้งส่วนขยายหรือล้างพื้นที่จัดเก็บข้อมูลส่วนขยายของ Chrome

3. การอนุญาตที่ใช้

การอนุญาตวัตถุประสงค์
storageจัดเก็บข้อมูลในเครื่องด้านบน (การตั้งค่า, ฉบับร่าง, ประวัติ, แพลตฟอร์มที่เลือก ฯลฯ) ภายในเบราว์เซอร์ของคุณ
offscreenเรียกใช้ ffmpeg.wasm ในเอกสาร offscreen เพื่อบีบอัด / เข้ารหัสไฟล์วิดีโอใหม่เมื่อแหล่งที่มาเกินขีดจำกัดขนาดของ SNS ปลายทาง ข้อมูลวิดีโอประมวลผลในเครื่องเท่านั้น; ไม่ได้ส่งไปที่ใดโดย Tutti
sidePanelโหมดการแสดงผล UI ทางเลือก: แสดงแบบฟอร์มสร้างโพสต์ของ Tutti ในแผงด้านข้างของ Chrome แทนที่ popup ของการดำเนินการ เพื่อให้เปิดอยู่ขณะที่คุณสลับระหว่างแท็บ SNS คุณเลือกใช้จากหน้าตัวเลือก; ค่าเริ่มต้นคือ popup มาตรฐาน
host_permissionsฉีด content scripts เข้าไปในแต่ละเว็บไซต์ SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) เพื่อกรอกแบบฟอร์มสร้างโพสต์ด้วยเนื้อหาโพสต์ของคุณและส่งในนามของคุณ ไม่มีการอ่านข้อมูลจากเว็บไซต์เหล่านี้และส่งไปที่อื่น; content script เขียนเฉพาะอินพุตของคุณเองเท่านั้น
optional_host_permissions: https://*/*ขอในขณะรันไทม์เฉพาะเมื่อคุณกำหนด URL อินสแตนซ์ Mastodon หรือ Misskey แบบกำหนดเอง (อื่นนอกจากค่าเริ่มต้น) Chrome แสดงพรอมต์การอนุญาตมาตรฐานในขณะนั้น

4. ความสัมพันธ์กับผู้ให้บริการ SNS

Tutti อัตโนมัติ Web UI ของ (หรือ ทางเลือก, โพสต์ผ่าน API อย่างเป็นทางการของ) 11 เครือข่ายที่รองรับ นี่เป็นฟังก์ชันที่ไม่เป็นทางการ; Tutti ไม่ได้มีความเกี่ยวข้องกับ X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok หรือ YouTube คุณยังคงผูกพันตามเงื่อนไขการใช้บริการของแต่ละบริการเอง
