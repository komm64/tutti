Tutti — Kebijakan privasi

Tidak ada pengumpulan data pihak ketiga. Semua konten postingan langsung dari browser Anda ke setiap SNS.

Beranda GitHub Dukungan

Terakhir diperbarui: 2026-05-23 (v0.5.0)

1. Data yang kami kumpulkan dan kirim

Tutti tidak memiliki server backend. Konten postingan (teks, gambar, video) tidak pernah dikirim ke server pihak ketiga mana pun. Konten ini berjalan langsung dari browser Anda ke setiap jejaring sosial yang Anda pilih, persis seolah-olah Anda mengetik dan mengirimnya sendiri di sana.

  • Teks, gambar, dan video postingan diteruskan hanya ke halaman SNS yang Anda pilih (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) di dalam browser Anda.
  • Saat jalur API resmi opsional diaktifkan untuk Bluesky / Mastodon / Misskey, postingan dikirim langsung dari browser Anda ke server API milik SNS tersebut menggunakan kredensial yang Anda berikan di Pengaturan. Tidak ada server Tutti / pengembang yang terlibat.
  • Tidak ada data analitik, pelacakan, atau iklan yang dikumpulkan.
  • Tutti tidak membaca atau menyimpan token autentikasi, cookie, atau data sesi Anda dari situs SNS.

Pengecualian — laporan kesalahan manual (persetujuan per klik)

Saat (dan hanya saat) Anda menekan tombol «Laporkan» yang ditampilkan di dialog kesalahan atau panel diagnosis, hal berikut dikirim melalui relay Cloudflare Workers ke pelacak issue GitHub pribadi (komm64/tutti-issues, hanya terlihat oleh pengembang Tutti):

  • Pesan kesalahan, versi ekstensi, string User-Agent browser.
  • 30 entri terakhir dari log internal ekstensi (tingkat log yang Anda konfigurasi; INFO secara default). Konten postingan, gambar, video, data login SNS, dan kredensial API tidak pernah disertakan.
  • Snapshot DOM yang disunting dari tab SNS tempat kesalahan terjadi, digunakan untuk memperbaiki selektor yang rusak. Konten teks, nilai atribut, jalur/kueri/fragmen URL, atribut href/src, dan tag <meta>/<link> dihapus. Payload dibatasi 8 KB. Lihat dom-snapshot.ts untuk implementasinya.

Tidak ada yang dikirim sampai Anda menekan tombol laporan. Tujuannya pribadi, hanya terlihat oleh pengembang Tutti.

2. Data yang disimpan secara lokal

Hal-hal berikut disimpan hanya di dalam browser Anda melalui API storage Chrome. Tidak ada yang dikirim keluar perangkat oleh Tutti.

  • Pengaturan (chrome.storage.sync): URL instance Mastodon / Misskey, sakelar autoPost, mode tampilan, URL pengganti selektor, dll. Dapat direplikasi ke perangkat lain Anda melalui Chrome Sync jika Anda masuk ke Chrome.
  • Draf (chrome.storage.session): Teks postingan yang sedang berlangsung dan media terlampir (gambar / video). Dihapus otomatis saat browser keluar.
  • SNS yang dipilih (chrome.storage.local): SNS yang Anda centang sebagai target postingan. Tetap ada antar sesi untuk kenyamanan; hanya lokal di perangkat.
  • Nama pengguna terakhir terlihat (chrome.storage.local): Nama pengguna yang saat ini masuk di setiap SNS, ditampilkan di popup sehingga Anda dapat mengonfirmasi akun mana yang akan menerima postingan. Hanya lokal di perangkat.
  • Riwayat postingan (chrome.storage.local): 20 postingan terakhir: 80 karakter pertama teks, platform target, keberhasilan/kegagalan per platform, dan stempel waktu. Hanya lokal di perangkat.
  • Kredensial API (chrome.storage.local): Hanya jika Anda secara eksplisit mengaktifkan jalur API resmi untuk Bluesky / Mastodon / Misskey: Bluesky app password atau token akses Mastodon / Misskey Anda. Hanya lokal di perangkat; tidak pernah direplikasi melalui Chrome Sync; tidak pernah dikirim ke server pengembang mana pun. Dapat dihapus dari wizard Pengaturan.
  • Cache pengganti selektor (chrome.storage.local): Data hot-fix selektor yang diambil dari URL yang Anda konfigurasi di Pengaturan (default: selectors.json publik di GitHub Pages Tutti).
  • Cache batas unggahan video (chrome.storage.local): Hasil probe API untuk batas unggahan video per akun (mis. Bluesky). Hanya nilai numerik; tidak ada kredensial.

Semua ini dapat dihapus kapan saja dengan menghapus instalasi ekstensi atau membersihkan penyimpanan ekstensi Chrome.

3. Izin yang digunakan

IzinTujuan
storageMenyimpan data lokal di atas (pengaturan, draf, riwayat, platform yang dipilih, dll.) di dalam browser Anda.
offscreenMenjalankan ffmpeg.wasm di dokumen offscreen untuk mengompres / mengkode ulang file video saat sumber melebihi batas ukuran SNS tujuan. Data video diproses hanya secara lokal; tidak dikirim ke mana pun oleh Tutti.
sidePanelMode tampilan UI opsional: menampilkan formulir penulisan Tutti di panel samping Chrome alih-alih popup tindakan, agar tetap terbuka saat Anda beralih antara tab SNS. Anda mengaktifkannya melalui halaman Opsi; default adalah popup standar.
host_permissionsMenyuntikkan content scripts ke setiap situs SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) untuk mengisi formulir penulisan dengan konten postingan Anda dan mengirimkannya atas nama Anda. Tidak ada data yang dibaca dari situs ini dan dikirim ke tempat lain; content script hanya menulis input Anda sendiri.
optional_host_permissions: https://*/*Diminta saat runtime hanya jika Anda mengonfigurasi URL instance Mastodon atau Misskey kustom (selain default). Chrome menampilkan permintaan izin standar pada saat itu.

4. Hubungan dengan penyedia SNS

Tutti mengotomatiskan UI web dari (atau, secara opsional, memposting melalui API resmi) 11 jejaring yang didukung. Ini adalah fungsi tidak resmi; Tutti tidak berafiliasi dengan X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok, atau YouTube. Anda tetap terikat oleh ketentuan penggunaan masing-masing layanan.
