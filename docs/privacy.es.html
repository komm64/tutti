Tutti — Política de privacidad

Sin recopilación de datos por terceros. Todo el contenido de publicaciones va directamente desde tu navegador a cada SNS.

Inicio GitHub Soporte

Última actualización: 2026-05-23 (v0.5.0)

1. Datos que recopilamos y transmitimos

Tutti no tiene servidor backend. El contenido de las publicaciones (texto, imágenes, vídeo) nunca se transmite a ningún servidor de terceros. Viaja directamente desde tu navegador a cada red social que selecciones, exactamente como si lo hubieras escrito y enviado allí tú mismo.

  • El texto, las imágenes y el vídeo de las publicaciones se pasan únicamente a las páginas de los SNS que elijas (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) dentro de tu navegador.
  • Cuando la ruta opcional de API oficial está habilitada para Bluesky / Mastodon / Misskey, las publicaciones se envían directamente desde tu navegador al servidor API propio de ese SNS utilizando las credenciales que proporcionaste en la configuración. No interviene ningún servidor de Tutti ni del desarrollador.
  • No se recopilan datos analíticos, de seguimiento ni publicitarios.
  • Tutti no lee ni almacena tus tokens de autenticación, cookies o datos de sesión de los sitios SNS.

Excepción — informes manuales de errores (consentimiento por clic)

Cuando (y solo cuando) pulsas el botón “Informar” mostrado en un cuadro de diálogo de error o en el panel de diagnóstico, se envía lo siguiente a través de un relay de Cloudflare Workers a un rastreador privado de incidencias de GitHub (komm64/tutti-issues, visible solo para el desarrollador de Tutti):

  • El mensaje de error, la versión de la extensión y la cadena User-Agent del navegador.
  • Las últimas 30 entradas del registro interno de la extensión (el nivel de log que hayas configurado; INFO por defecto). El contenido de las publicaciones, las imágenes, el vídeo, los datos de inicio de sesión en SNS y las credenciales de API nunca se incluyen.
  • Una instantánea DOM redactada de la pestaña SNS donde ocurrió el error, utilizada para reparar selectores rotos. El contenido de texto, los valores de atributos, las rutas/consultas/fragmentos de URL, los atributos href/src y las etiquetas <meta>/<link> se eliminan. El payload está limitado a 8 KB. Consulta dom-snapshot.ts para la implementación.

Nada se envía hasta que pulses el botón de informe. El destino es privado, visible solo para el desarrollador de Tutti.

2. Datos almacenados localmente

Lo siguiente se almacena solo dentro de tu navegador a través de la API storage de Chrome. Tutti no transmite nada fuera del dispositivo.

  • Configuración (chrome.storage.sync): URLs de instancias de Mastodon / Misskey, conmutador autoPost, modo de visualización, URL de override de selectores, etc. Puede replicarse a tus otros dispositivos mediante Chrome Sync si has iniciado sesión en Chrome.
  • Borradores (chrome.storage.session): Texto de publicación en progreso y medios adjuntos (imágenes / vídeo). Se borra automáticamente al cerrar el navegador.
  • SNS seleccionados (chrome.storage.local): Los SNS que marcaste como destinos de publicación. Persiste entre sesiones por comodidad; solo local al dispositivo.
  • Últimos nombres de usuario vistos (chrome.storage.local): El nombre de usuario actualmente conectado en cada SNS, mostrado en el popup para que puedas confirmar qué cuenta recibirá la publicación. Solo local al dispositivo.
  • Historial de publicaciones (chrome.storage.local): Las últimas 20 publicaciones: primeros 80 caracteres de texto, plataformas objetivo, éxito/fallo por plataforma y marca de tiempo. Solo local al dispositivo.
  • Credenciales API (chrome.storage.local): Solo si habilitaste explícitamente la ruta de API oficial para Bluesky / Mastodon / Misskey: tu app password de Bluesky o token de acceso de Mastodon / Misskey. Solo local al dispositivo; nunca se replica vía Chrome Sync; nunca se transmite a ningún servidor del desarrollador. Eliminable desde el asistente de configuración.
  • Caché de override de selectores (chrome.storage.local): Datos de hot-fix de selectores obtenidos de la URL que configuraste en la configuración (por defecto: el selectors.json público en las páginas de GitHub de Tutti).
  • Caché de límite de carga de vídeo (chrome.storage.local): Resultado de sondeos API para límites de carga de vídeo por cuenta (p. ej., Bluesky). Solo valores numéricos; sin credenciales.

Todo esto puede borrarse en cualquier momento desinstalando la extensión o limpiando el almacenamiento de extensiones de Chrome.

3. Permisos utilizados

PermisoPropósito
storageAlmacenar los datos locales anteriores (configuración, borradores, historial, plataformas seleccionadas, etc.) dentro de tu navegador.
offscreenEjecutar ffmpeg.wasm en un documento offscreen para comprimir / recodificar archivos de vídeo cuando el origen exceda el límite de tamaño del SNS de destino. Los datos de vídeo se procesan solo localmente; Tutti no los transmite a ninguna parte.
sidePanelModo de visualización de UI opcional: muestra el formulario de redacción de Tutti en el panel lateral de Chrome en lugar del popup de acción, para que permanezca abierto mientras cambias entre pestañas de SNS. Optas por él desde la página de Opciones; el valor predeterminado es el popup estándar.
host_permissionsInyectar content scripts en cada sitio SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) para rellenar el formulario de redacción con el contenido de tu publicación y enviarlo en tu nombre. No se leen datos de estos sitios para enviarlos a otra parte; el content script solo escribe tu propia entrada.
optional_host_permissions: https://*/*Solicitado en tiempo de ejecución solo si configuras una URL de instancia personalizada de Mastodon o Misskey (distinta de las predeterminadas). Chrome muestra el aviso de permiso estándar en ese momento.

4. Relación con los proveedores de SNS

Tutti automatiza la UI web de (u opcionalmente publica a través de la API oficial de) las 11 redes compatibles. Esta es una funcionalidad no oficial; Tutti no está afiliado a X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok ni YouTube. Sigues sujeto a los términos de uso propios de cada servicio.
