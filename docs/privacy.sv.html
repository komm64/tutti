Tutti — Integritetspolicy

Ingen datainsamling av tredje part. Allt inläggsinnehåll går direkt från din webbläsare till varje SNS.

Hem GitHub Support

Senast uppdaterad: 2026-05-23 (v0.5.0)

1. Data vi samlar in och överför

Tutti har ingen backend-server. Inläggsinnehåll (text, bilder, video) överförs aldrig till någon tredjepartsserver. Det färdas direkt från din webbläsare till varje socialt nätverk du väljer, precis som om du hade skrivit och skickat det där själv.

  • Inläggets text, bilder och video skickas endast till de SNS-sidor du väljer (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) inom din webbläsare.
  • När den valfria officiella API-vägen är aktiverad för Bluesky / Mastodon / Misskey skickas inlägg direkt från din webbläsare till det SNS:s egna API-server med de uppgifter du angett i Inställningar. Ingen Tutti- / utvecklarserver är inblandad.
  • Inga analys-, spårnings- eller annonsdata samlas in.
  • Tutti läser inte och lagrar inte dina autentiseringstokens, cookies eller sessionsdata från SNS-webbplatserna.

Undantag — manuella felrapporter (samtycke per klick)

När (och endast när) du trycker på knappen “Rapportera” som visas i en feldialog eller diagnospanelen skickas följande via ett Cloudflare Workers-relä till en privat GitHub-issue-tracker (komm64/tutti-issues, endast synlig för Tutti-utvecklaren):

  • Felmeddelandet, tilläggets version och webbläsarens User-Agent-sträng.
  • De senaste 30 posterna i tilläggets interna logg (loggnivån du konfigurerat; INFO som standard). Inläggsinnehåll, bilder, video, SNS-inloggningsdata och API-uppgifter ingår aldrig.
  • En redigerad DOM-ögonblicksbild av SNS-fliken där felet inträffade, används för att reparera trasiga selektorer. Textinnehåll, attributvärden, URL-sökvägar/-frågor/-fragment, href/src-attribut och <meta>/<link>-taggar tas bort. Nyttolasten är begränsad till 8 KB. Se dom-snapshot.ts för implementeringen.

Inget skickas förrän du trycker på rapporteringsknappen. Destinationen är privat och endast synlig för Tutti-utvecklaren.

2. Lokalt lagrade data

Följande lagras endast i din webbläsare via Chromes storage-API. Inget överförs från enheten av Tutti.

  • Inställningar (chrome.storage.sync): Mastodon-/Misskey-instans-URL:er, autoPost-omkopplare, visningsläge, selector-override-URL osv. Kan replikeras till dina andra enheter via Chrome Sync om du är inloggad på Chrome.
  • Utkast (chrome.storage.session): Pågående inläggstext och bifogade media (bilder / video). Rensas automatiskt när webbläsaren avslutas.
  • Valda SNS (chrome.storage.local): De SNS du markerat som publiceringsmål. Bibehålls mellan sessioner för bekvämlighets skull; endast lokalt på enheten.
  • Senast sedda användarnamn (chrome.storage.local): Det användarnamn som för närvarande är inloggat på varje SNS, visat i popupen så att du kan bekräfta vilket konto som tar emot inlägget. Endast lokalt på enheten.
  • Inläggshistorik (chrome.storage.local): De senaste 20 inläggen: de första 80 tecknen av texten, målplattformar, framgång/misslyckande per plattform och tidsstämpel. Endast lokalt på enheten.
  • API-uppgifter (chrome.storage.local): Endast om du uttryckligen aktiverat den officiella API-vägen för Bluesky / Mastodon / Misskey: ditt Bluesky app password eller Mastodon-/Misskey-access-token. Endast lokalt på enheten; replikeras aldrig via Chrome Sync; skickas aldrig till någon utvecklarserver. Borttagbart från inställningsguiden.
  • Cache för selector-override (chrome.storage.local): Selector-hot-fix-data hämtad från den URL du konfigurerat i Inställningar (standard: den offentliga selectors.json på Tuttis GitHub Pages).
  • Cache för videouppladdningsgräns (chrome.storage.local): Resultat av API-sonderingar för videouppladdningsgränser per konto (t.ex. Bluesky). Endast numeriska värden; inga uppgifter.

Allt detta kan raderas när som helst genom att avinstallera tillägget eller rensa Chromes tilläggslagring.

3. Använda behörigheter

BehörighetSyfte
storageLagra ovanstående lokala data (inställningar, utkast, historik, valda plattformar osv.) i din webbläsare.
offscreenKöra ffmpeg.wasm i ett offscreen-dokument för att komprimera / omkoda videofiler när källan överskrider storleksgränsen för mål-SNS. Videodata bearbetas endast lokalt; överförs inte någonstans av Tutti.
sidePanelValfritt UI-visningsläge: visar Tuttis komposeringsformulär i Chromes sidopanel istället för åtgärds-popupen, så att det förblir öppet medan du växlar mellan SNS-flikar. Du väljer detta från Alternativ-sidan; standardvärdet är standard-popupen.
host_permissionsInjicera content scripts i varje SNS-webbplats (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) för att fylla i komposeringsformuläret med ditt inläggsinnehåll och skicka det å dina vägnar. Inga data läses från dessa webbplatser och skickas någon annanstans; content-skriptet skriver endast din egen inmatning.
optional_host_permissions: https://*/*Begärs vid körning endast om du konfigurerar en anpassad Mastodon- eller Misskey-instans-URL (annan än standardvärdena). Chrome visar standardbehörighetsprompten vid den tidpunkten.

4. Förhållande till SNS-leverantörerna

Tutti automatiserar webb-UI:n (eller, valfritt, publicerar via det officiella API:et) för de 11 stödda nätverken. Detta är inofficiell funktionalitet; Tutti är inte ansluten till X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok eller YouTube. Du förblir bunden av varje tjänsts egna användarvillkor.
