Tutti — 개인정보 처리방침

제3자 데이터 수집 없음. 모든 게시 콘텐츠는 브라우저에서 각 SNS로 직접 전송됩니다.

홈 GitHub 지원

최종 업데이트: 2026-05-23 (v0.5.0)

1. 수집·전송하는 데이터

Tutti에는 백엔드 서버가 없습니다. 게시 콘텐츠 (텍스트, 이미지, 동영상)는 어떠한 제3자 서버에도 전송되지 않습니다. 마치 귀하가 직접 입력하고 제출한 것처럼, 브라우저에서 선택한 각 소셜 네트워크로 직접 전달됩니다.

  • 게시 텍스트, 이미지, 동영상은 귀하가 선택한 SNS 페이지 (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube)에 브라우저 내에서만 전달됩니다.
  • Bluesky / Mastodon / Misskey에 대해 선택적 공식 API 경로를 활성화한 경우, 설정에서 제공한 자격 증명을 사용하여 게시물이 브라우저에서 해당 SNS 자체 API 서버로 직접 전송됩니다. Tutti / 개발자 서버는 관여하지 않습니다.
  • 분석, 추적 또는 광고 데이터를 수집하지 않습니다.
  • Tutti는 SNS 사이트의 인증 토큰, 쿠키, 세션 데이터를 읽거나 저장하지 않습니다.

예외 — 수동 오류 보고 (클릭 시마다 동의)

오류 대화 상자 또는 진단 패널에 표시된 「보고」 버튼을 누르신 경우 (그리고 그때만), 다음 항목이 Cloudflare Workers 릴레이를 통해 비공개 GitHub issue tracker (komm64/tutti-issues, Tutti 개발자만 열람 가능)로 전송됩니다:

  • 오류 메시지, 확장 프로그램 버전, 브라우저 User-Agent 문자열.
  • 확장 프로그램의 내부 로그 마지막 30개 (구성한 로그 수준; 기본 INFO). 게시 콘텐츠, 이미지, 동영상, SNS 로그인 데이터, API 자격 증명은 절대 포함되지 않습니다.
  • 오류가 발생한 SNS 탭의 redacted DOM 스냅샷. 깨진 선택자를 복구하는 데 사용됩니다. 텍스트 내용, 속성값, URL 경로/쿼리/조각, href/src 속성, <meta>/<link> 태그는 모두 제거됩니다. 페이로드는 8 KB로 제한됩니다. 구현은 dom-snapshot.ts를 참조하세요.

보고 버튼을 누르기 전까지는 아무것도 전송되지 않습니다. 수신처는 비공개이며 Tutti 개발자만 열람 가능합니다.

2. 로컬에 저장되는 데이터

다음 항목은 Chrome의 storage API를 통해 귀하의 브라우저 내에만 저장됩니다. Tutti가 외부로 전송하는 일은 없습니다.

  • 설정 (chrome.storage.sync): Mastodon / Misskey 인스턴스 URL, autoPost 토글, 표시 방식, selector override URL 등. Chrome에 로그인되어 있는 경우 Chrome Sync를 통해 다른 기기로 복제될 수 있습니다.
  • 초안 (chrome.storage.session): 작성 중인 게시 텍스트 및 첨부 미디어 (이미지 / 동영상). 브라우저 종료 시 자동 삭제.
  • 선택된 SNS (chrome.storage.local): 게시 대상으로 체크한 SNS. 편의를 위해 세션 간에 유지됩니다; 기기 로컬만.
  • 마지막으로 확인된 사용자명 (chrome.storage.local): 각 SNS의 현재 로그인된 사용자명. 어느 계정이 게시물을 받을지 확인할 수 있도록 popup에 표시됩니다. 기기 로컬만.
  • 게시 기록 (chrome.storage.local): 최근 20개의 게시물: 텍스트의 처음 80자, 대상 플랫폼, 플랫폼별 성공/실패 상태, 타임스탬프. 기기 로컬만.
  • API 자격 증명 (chrome.storage.local): Bluesky / Mastodon / Misskey의 공식 API 경로를 명시적으로 활성화한 경우에만 저장: 귀하의 Bluesky app password 또는 Mastodon / Misskey access token. 기기 로컬만; Chrome Sync로 복제되지 않음; 어떠한 개발자 서버에도 전송되지 않음. 설정 마법사에서 제거할 수 있습니다.
  • 선택자 오버라이드 캐시 (chrome.storage.local): 설정에서 구성한 URL에서 가져온 선택자 핫픽스 데이터 (기본값: Tutti의 GitHub Pages에 있는 공개 selectors.json).
  • 동영상 업로드 한도 캐시 (chrome.storage.local): 계정별 동영상 업로드 한도 (예: Bluesky)에 대한 API 프로브 결과. 숫자 값만; 자격 증명 없음.

확장 프로그램을 제거하거나 Chrome의 확장 프로그램 저장소를 지우면 언제든지 모두 완전히 삭제할 수 있습니다.

3. 사용 권한

권한용도
storage위의 로컬 데이터 (설정, 초안, 기록, 선택된 플랫폼 등)를 브라우저 내에 저장하기 위해.
offscreen소스가 대상 SNS의 크기 제한을 초과할 때 offscreen 문서에서 ffmpeg.wasm을 실행하여 동영상 파일을 압축 / 재인코딩하기 위해. 동영상 데이터는 로컬에서만 처리됩니다; Tutti가 어디로도 전송하지 않습니다.
sidePanel선택적 UI 표시 방식: 액션 popup 대신 Chrome의 사이드 패널에 Tutti의 작성 양식을 표시하여 SNS 탭 사이를 전환할 때에도 열린 상태를 유지합니다. 옵션 페이지에서 사용 설정합니다; 기본값은 표준 popup입니다.
host_permissions각 SNS 사이트 (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com)에 content script를 주입하여 작성 양식에 귀하 대신 게시 콘텐츠를 채우고 제출하기 위해. 이 사이트에서 데이터를 읽어 다른 곳으로 보내지 않습니다; content script는 귀하 본인의 입력만 작성합니다.
optional_host_permissions: https://*/*사용자 지정 Mastodon 또는 Misskey 인스턴스 URL (기본값 외)을 구성하는 경우에만 런타임에 요청됩니다. 그 순간 Chrome이 표준 권한 프롬프트를 표시합니다.

4. SNS 제공업체와의 관계

Tutti는 11개의 지원 네트워크의 웹 UI를 자동화 (또는 선택적으로 공식 API를 통해 게시)합니다. 이는 비공식 기능이며, Tutti는 X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok 또는 YouTube와 제휴 관계가 없습니다. 귀하는 여전히 각 서비스 자체의 이용 약관에 따라야 합니다.
