Tutti — Политика конфиденциальности

Никакого сбора данных третьими сторонами. Всё содержимое постов идёт прямо из вашего браузера в каждую SNS.

Главная GitHub Поддержка

Последнее обновление: 2026-05-23 (v0.5.0)

1. Собираемые и передаваемые данные

У Tutti нет бэкенд-сервера. Содержимое постов (текст, изображения, видео) никогда не передаётся ни одному стороннему серверу. Оно идёт напрямую из вашего браузера в каждую выбранную вами соцсеть, точно так же, как если бы вы сами набрали и отправили его там.

  • Текст, изображения и видео постов передаются только на выбранные вами страницы SNS (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) внутри вашего браузера.
  • Когда включён опциональный путь через официальный API для Bluesky / Mastodon / Misskey, посты отправляются напрямую из вашего браузера на собственный API-сервер этой SNS с использованием учётных данных, которые вы предоставили в Настройках. Никакой сервер Tutti / разработчика не задействован.
  • Никакие аналитические, трекинговые или рекламные данные не собираются.
  • Tutti не читает и не хранит ваши токены аутентификации, куки или данные сессии с сайтов SNS.

Исключение — ручные отчёты об ошибках (согласие на каждый клик)

Когда (и только когда) вы нажимаете кнопку «Сообщить», показанную в диалоге ошибки или панели диагностики, через релей Cloudflare Workers в приватный трекер задач GitHub (komm64/tutti-issues, видимый только разработчику Tutti) отправляется следующее:

  • Сообщение об ошибке, версия расширения, строка User-Agent браузера.
  • Последние 30 записей внутреннего журнала расширения (настроенный вами уровень лога; по умолчанию INFO). Содержимое постов, изображения, видео, данные входа в SNS и учётные данные API никогда не включаются.
  • Отредактированный DOM-снимок вкладки SNS, где произошла ошибка, используемый для починки сломанных селекторов. Текстовое содержимое, значения атрибутов, пути/запросы/фрагменты URL, атрибуты href/src и теги <meta>/<link> удаляются. Размер полезной нагрузки ограничен 8 КБ. Реализацию см. в dom-snapshot.ts.

Ничего не отправляется, пока вы не нажмёте кнопку отчёта. Назначение приватное, видимое только разработчику Tutti.

2. Локально сохраняемые данные

Следующее сохраняется только внутри вашего браузера через API storage Chrome. Tutti ничего не передаёт за пределы устройства.

  • Настройки (chrome.storage.sync): URL-адреса инстансов Mastodon / Misskey, переключатель autoPost, режим отображения, URL переопределения селекторов и т. д. Может реплицироваться на ваши другие устройства через Chrome Sync, если вы вошли в Chrome.
  • Черновики (chrome.storage.session): Текст поста в процессе и прикреплённые медиа (изображения / видео). Автоматически очищается при закрытии браузера.
  • Выбранные SNS (chrome.storage.local): SNS, которые вы отметили как цели публикации. Сохраняется между сессиями для удобства; только локально на устройстве.
  • Последние увиденные имена пользователей (chrome.storage.local): Текущее имя пользователя, вошедшего в каждую SNS, отображается во всплывающем окне, чтобы вы могли подтвердить, какой аккаунт получит пост. Только локально на устройстве.
  • История постов (chrome.storage.local): Последние 20 постов: первые 80 символов текста, целевые платформы, успех/неудача по платформе и временная метка. Только локально на устройстве.
  • Учётные данные API (chrome.storage.local): Только если вы явно включили путь через официальный API для Bluesky / Mastodon / Misskey: ваш Bluesky app password или токен доступа Mastodon / Misskey. Только локально на устройстве; никогда не реплицируется через Chrome Sync; никогда не передаётся ни на какой сервер разработчика. Удаляется из мастера Настроек.
  • Кэш переопределения селекторов (chrome.storage.local): Данные горячего исправления селекторов, полученные с URL, настроенного вами в Настройках (по умолчанию: публичный selectors.json на GitHub Pages Tutti).
  • Кэш предела загрузки видео (chrome.storage.local): Результат API-зондов для предельных значений загрузки видео по аккаунту (например, Bluesky). Только числовые значения; без учётных данных.

Всё это можно стереть в любое время, удалив расширение или очистив хранилище расширений Chrome.

3. Используемые разрешения

РазрешениеНазначение
storageХранить указанные выше локальные данные (настройки, черновики, история, выбранные платформы и т. д.) внутри вашего браузера.
offscreenЗапускать ffmpeg.wasm в офскрин-документе для сжатия / повторного кодирования видеофайлов, когда источник превышает ограничение размера целевой SNS. Видеоданные обрабатываются только локально; Tutti не передаёт их никуда.
sidePanelОпциональный режим отображения UI: показывает форму создания поста Tutti в боковой панели Chrome вместо всплывающего окна действия, чтобы она оставалась открытой при переключении между вкладками SNS. Вы включаете это на странице Параметров; по умолчанию используется обычное всплывающее окно.
host_permissionsВнедрять контент-скрипты на каждый сайт SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com), чтобы заполнить форму создания поста содержимым вашего поста и отправить его от вашего имени. Никакие данные не считываются с этих сайтов и не отправляются куда-либо ещё; контент-скрипт записывает только ваш собственный ввод.
optional_host_permissions: https://*/*Запрашивается во время выполнения только если вы настраиваете пользовательский URL инстанса Mastodon или Misskey (отличный от значений по умолчанию). Chrome показывает стандартный запрос разрешения в этот момент.

4. Отношения с провайдерами SNS

Tutti автоматизирует веб-интерфейс (или, опционально, публикует через официальный API) 11 поддерживаемых сетей. Это неофициальная функциональность; Tutti не аффилирован с X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok или YouTube. Вы по-прежнему обязаны соблюдать собственные условия использования каждого сервиса.
