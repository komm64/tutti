最終更新: 2026-05-23 (v0.5.0)

1. 取得・送信するデータ

Tutti にバックエンドサーバーは無く、 投稿コンテンツ(テキスト・画像・動画)はいかなる第三者サーバーにも送信されません。 ブラウザから直接、 ユーザーが選択した各 SNS に渡されます。

  • 投稿テキスト・画像・動画は、 ユーザーが選択した各 SNS のページに対し、 ブラウザ内でのみ渡されます。
  • Bluesky / Mastodon / Misskey で公式 API 経路を有効にしている場合、 Settings に登録された認証情報を使って 各 SNS の公式 API サーバーへ直接 接続します(開発者サーバーは介在しません)。
  • 解析・トラッキング・広告のためのデータ収集は行いません。
  • SNS の認証 token / cookie / セッション情報を Tutti が読み取り・保存することは ありません。

例外 — 手動エラー報告 (click ごとに opt-in)

popup のエラー dialog または診断 panel にある 「報告する」 button を user が押した時だけ、 以下が Cloudflare Workers 経由で private な GitHub issue tracker (komm64/tutti-issues、 Tutti 開発者のみ閲覧可) に送信されます:

  • エラーメッセージ、 拡張バージョン、 ブラウザ User-Agent 文字列。
  • 拡張内部ログの末尾 30 件 (設定された log level、 default INFO)。 投稿の本文・画像・動画・SNS のログイン情報・API credentials は 含まれません。
  • セレクタ修復用に、 操作対象 SNS タブの redacted DOM snapshot。 text / 属性値 / URL の path / query / fragment、 href / src 属性、 <meta> / <link> タグを削除した上で、 最大 8 KB で送信されます。 詳細は dom-snapshot.ts を参照。

報告 button を押すまで送信は発生しません。 送信先は private、 Tutti 開発者本人のみ閲覧できます。

注 (v0.4.32-v0.4.34 の経緯): 旧バージョンでは報告先が public な GitHub Issues でしたが、 redaction layer の bug で意図しない情報 (閲覧中の YouTube 動画 URL 等) が公開状態に出る事故が発生しました。 v0.4.35 以降は private repo に切り替え、 万が一 redaction が破れても外部からは見えない構造になっています。

2. ローカルに保存するデータ

Chrome の storage API でユーザーのブラウザ内にのみ保存されます。 Tutti から外部に送信されることはありません。

  • 設定 (chrome.storage.sync): Mastodon / Misskey インスタンス URL、 autoPost toggle、 表示方式 (popup / sidepanel / floating)、 Selector override URL 等。 Chrome 同期で他デバイスにも複製される場合あり。
  • 下書き (chrome.storage.session): 入力中のテキスト + 添付メディア (画像・動画)。 ブラウザ終了で自動消去。
  • SNS 選択 (chrome.storage.local): 投稿先としてチェックした SNS。 投稿後もリセットされません。 デバイスローカルのみ。
  • ログイン中アカウント名 (chrome.storage.local): 各 SNS の現在ログイン中ユーザー名 (誤投稿防止のため popup に表示)。 デバイスローカルのみ。
  • 投稿履歴 (chrome.storage.local): 直近 20 件の先頭 80 文字、 対象プラットフォーム、 成否、 タイムスタンプ。 デバイスローカルのみ。
  • API credentials (chrome.storage.local): Bluesky / Mastodon / Misskey の公式 API 経路を user が明示的に有効にした場合のみ保存される、 Bluesky app password または Mastodon / Misskey の access token。 デバイスローカル限定。 Chrome Sync には載らず、 開発者サーバーにも送信されません。 Settings wizard から個別削除可。
  • Selector override キャッシュ (chrome.storage.local): Settings で指定された URL から取得した selector 上書きデータ (default は Tutti GitHub Pages の公式 selectors.json)。
  • 動画上限キャッシュ (chrome.storage.local): Bluesky 等の動画 upload 上限を API probe した結果。 数値のみ、 認証情報を含みません。

拡張のアンインストール or Chrome の拡張データクリアでいつでも完全削除できます。

3. 必要な権限

権限用途
storage上記のローカルデータ (設定・下書き・履歴・選択 SNS 等) をブラウザ内に保存するため。
offscreen動画ファイルが投稿先 SNS の上限を超える時、 ffmpeg.wasm でブラウザ内 (offscreen document) で再エンコード (圧縮) するため。 動画データはローカル処理のみで Tutti が外部送信することはありません。
sidePanel表示方式のオプション: Tutti の compose form を Chrome の side panel に表示する選択肢。 SNS タブを行き来しても閉じない UX を提供。 Options で user が明示選択、 default は標準 popup。
host_permissions各 SNS サイト (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) に content script を inject して、 compose form に投稿内容を埋めて submit するため。 これらのサイトからデータを読み取り外部送信することはありません。 content script は user 入力を form に書き込むだけです。
optional_host_permissions: https://*/*user が Mastodon / Misskey の代替インスタンスを設定した時のみ動的に Chrome の許可 dialog を出します。 それ以外には使いません。

4. 外部サービスとの関係

Tutti は対応 11 ネットワークの Web 画面を自動操作 (or 公式 API 経由で投稿) します。 これらは公式機能ではなく、 X / Bluesky / Meta (Threads / Instagram) / Mastodon gGmbH / Misskey / Tumblr / Pixiv / DeviantArt / TikTok / YouTube とは無関係です。 各サービスの利用規約は各サービスのものに従ってください。
