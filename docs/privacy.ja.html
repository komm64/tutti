Tutti — プライバシーポリシー

第三者サーバーを経由しません。投稿内容はブラウザから各 SNS に直接渡されます。

ホーム GitHub サポート

最終更新: 2026-05-23 (v0.5.0)

1. 取得・送信するデータ

Tutti にバックエンドサーバーは存在しません。 投稿コンテンツ (テキスト・画像・動画) はいかなる第三者サーバーにも送信されません。ブラウザから直接、ユーザーが選択した各 SNS に、自分でそこに入力して送信したのと同じように渡されます。

  • 投稿テキスト・画像・動画は、ユーザーが選択した SNS (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) のページにブラウザ内でのみ渡されます。
  • Bluesky / Mastodon / Misskey で公式 API 経路を有効にしている場合、Settings に登録された認証情報を使って 各 SNS の公式 API サーバーへ直接 接続します。Tutti / 開発者サーバーは介在しません。
  • 解析・トラッキング・広告のためのデータ収集は行いません。
  • Tutti は SNS の認証 token / cookie / セッションデータを 読み取りません。保存もしません。

例外 — 手動エラー報告 (click ごとに opt-in)

popup のエラー dialog または診断 panel に表示される 「報告する」ボタンをユーザーが押した時 (またその時に限り)、以下が Cloudflare Workers リレー経由で private な GitHub issue tracker (komm64/tutti-issues、Tutti 開発者のみ閲覧可) に送信されます:

  • エラーメッセージ、拡張バージョン、ブラウザ User-Agent 文字列。
  • 拡張内部ログの末尾 30 件 (設定された log level、default は INFO)。投稿本文・画像・動画・SNS のログイン情報・API credentials は 含まれません。
  • セレクタ修復用に、エラーが発生した SNS タブの redacted DOM snapshot。テキスト・属性値・URL の path/query/fragment、href/src 属性、<meta>/<link> タグを削除した上で、最大 8 KB で送信されます。実装は dom-snapshot.ts を参照。

報告ボタンを押すまで送信は発生しません。送信先は private で、Tutti 開発者のみ閲覧できます。

2. ローカルに保存するデータ

以下は Chrome の storage API を介してユーザーのブラウザ内にのみ保存されます。Tutti から外部に送信されることはありません。

  • 設定 (chrome.storage.sync): Mastodon / Misskey インスタンス URL、autoPost トグル、表示方式、Selector override URL 等。Chrome にサインインしていれば Chrome Sync で他デバイスに複製される場合があります。
  • 下書き (chrome.storage.session): 入力中のテキスト + 添付メディア (画像・動画)。ブラウザ終了時に自動消去。
  • SNS 選択 (chrome.storage.local): 投稿先としてチェックした SNS。便宜上セッションをまたいで保持されます。デバイスローカルのみ。
  • ログイン中アカウント名 (chrome.storage.local): 各 SNS の現在ログイン中ユーザー名。どのアカウントに投稿されるかを popup で確認できるよう表示されます。デバイスローカルのみ。
  • 投稿履歴 (chrome.storage.local): 直近 20 件の投稿。先頭 80 文字・対象プラットフォーム・プラットフォーム別の成否・タイムスタンプを記録。デバイスローカルのみ。
  • API credentials (chrome.storage.local): Bluesky / Mastodon / Misskey の公式 API 経路を明示的に有効にした場合のみ保存される、Bluesky app password または Mastodon / Misskey の access token。デバイスローカル限定。Chrome Sync には載らず、開発者サーバーにも送信されません。 Settings の wizard から個別削除可能。
  • Selector override キャッシュ (chrome.storage.local): Settings で指定された URL から取得した selector hot-fix データ (default は Tutti GitHub Pages の公式 selectors.json)。
  • 動画 upload 上限キャッシュ (chrome.storage.local): Bluesky 等の動画 upload 上限を API probe した結果。数値のみ、認証情報を含みません。

拡張のアンインストール、または Chrome の拡張ストレージクリアでいつでもすべて完全削除できます。

3. 必要な権限

権限用途
storage上記のローカルデータ (設定・下書き・履歴・選択 SNS 等) をブラウザ内に保存するため。
offscreen動画ファイルが投稿先 SNS のサイズ上限を超える場合、ffmpeg.wasm を offscreen document 内で実行し、圧縮・再エンコードするため。動画データはローカル処理のみで、Tutti から外部送信されることはありません。
sidePanelオプションの UI 表示方式: Tutti の投稿フォームを通常の popup ではなく Chrome の side panel に表示します。SNS タブを切り替えても開いたまま使えます。Options ページで明示的に opt-in、default は標準の popup です。
host_permissions各 SNS サイト (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) に content script を inject し、投稿フォームに本文を入力してユーザーに代わって送信するため。これらのサイトからデータを読み取って外部送信することはありません。content script はユーザーの入力を書き込むだけです。
optional_host_permissions: https://*/*Mastodon / Misskey の代替インスタンス URL (default 以外) を設定した時のみ runtime で要求されます。その時点で Chrome の標準的な権限ダイアログが表示されます。

4. SNS 各社との関係

Tutti は対応 11 ネットワークの Web UI を自動操作 (または公式 API 経由で投稿) します。これは非公式機能であり、Tutti は X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok, YouTube とは無関係です。各サービスの利用規約には引き続き従ってください。
