Tutti — Adatvédelmi szabályzat

Nincs harmadik féltől származó adatgyűjtés. Minden bejegyzéstartalom közvetlenül a böngészőjéből megy minden SNS-hez.

Kezdőlap GitHub Támogatás

Utoljára frissítve: 2026-05-23 (v0.5.0)

1. Az általunk gyűjtött és továbbított adatok

A Tutti nem rendelkezik háttérszerverrel. A bejegyzések tartalma (szöveg, képek, videó) soha nem kerül továbbításra harmadik fél szerverére. Közvetlenül a böngészőjéből utazik minden kiválasztott közösségi hálózathoz, pontosan úgy, mintha Ön maga írta és küldte volna be ott.

  • A bejegyzés szövege, képei és videója csak az Ön által kiválasztott SNS oldalakra (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) kerül továbbításra a böngészőjén belül.
  • Ha a választható hivatalos API útvonal engedélyezve van a Bluesky / Mastodon / Misskey számára, a bejegyzések közvetlenül a böngészőjéből kerülnek küldésre az adott SNS saját API szerverére a Beállításokban megadott hitelesítő adatok felhasználásával. Semmilyen Tutti / fejlesztői szerver nincs bevonva.
  • Nem gyűjtünk semmilyen analitikai, követési vagy hirdetési adatot.
  • A Tutti nem olvas és nem tárol semmilyen hitelesítési tokent, sütit vagy munkamenet-adatot az SNS oldalakról.

Kivétel — kézi hibajelentések (kattintásonkénti hozzájárulás)

Amikor (és csak akkor) megnyomja a hibapárbeszédpanelben vagy a diagnosztikai panelen megjelenő „Jelentés” gombot, a következőket küldjük el Cloudflare Workers reléjen keresztül egy privát GitHub issue trackerbe (komm64/tutti-issues, amelyet csak a Tutti fejlesztője láthat):

  • A hibaüzenet, a bővítmény verziója, a böngésző User-Agent karakterlánca.
  • A bővítmény belső naplójának utolsó 30 bejegyzése (az Ön által konfigurált naplószint; alapértelmezésben INFO). A bejegyzés tartalma, képek, videó, SNS bejelentkezési adatok és API hitelesítő adatok soha nem szerepelnek benne.
  • A hibát okozó SNS lap szerkesztett DOM pillanatfelvétele, amelyet a sérült szelektorok javítására használunk. A szöveges tartalom, attribútumértékek, URL útvonalak/lekérdezések/töredékek, href/src attribútumok és <meta>/<link> tagek eltávolításra kerülnek. A hasznos teher 8 KB-ra van korlátozva. Az implementációhoz lásd a dom-snapshot.ts fájlt.

Semmi nem kerül küldésre, amíg meg nem nyomja a jelentési gombot. A célhely privát, csak a Tutti fejlesztője láthatja.

2. Helyileg tárolt adatok

A következő adatok csak a böngészőjén belül kerülnek tárolásra a Chrome storage API-ján keresztül. A Tutti semmit sem továbbít az eszközről.

  • Beállítások (chrome.storage.sync): Mastodon / Misskey példány URL-ek, autoPost kapcsoló, megjelenítési mód, szelektor felülíró URL stb. Chrome Synccel replikálható más eszközeire, ha bejelentkezett Chrome-ba.
  • Piszkozatok (chrome.storage.session): Folyamatban lévő bejegyzésszöveg és csatolt média (képek / videó). A böngésző bezárásakor automatikusan törlődik.
  • Kiválasztott SNS-ek (chrome.storage.local): Az SNS-ek, amelyeket bejegyzéscélokként bejelölt. Munkamenetek között megmarad a kényelem érdekében; csak helyileg az eszközön.
  • Utoljára látott felhasználónevek (chrome.storage.local): Az egyes SNS-eken jelenleg bejelentkezett felhasználónév, amely a popup-ban jelenik meg, hogy megerősítse, melyik fiók kapja a bejegyzést. Csak helyileg az eszközön.
  • Bejegyzés-előzmények (chrome.storage.local): Az utolsó 20 bejegyzés: a szöveg első 80 karaktere, célplatformok, sikeresség/sikertelenség platformonként és időbélyeg. Csak helyileg az eszközön.
  • API hitelesítő adatok (chrome.storage.local): Csak ha kifejezetten engedélyezte a hivatalos API útvonalat a Bluesky / Mastodon / Misskey számára: a Bluesky app password vagy Mastodon / Misskey hozzáférési token. Csak helyileg az eszközön; soha nem replikálódik Chrome Sync-en keresztül; soha nem továbbítódik semmilyen fejlesztői szerverre. A Beállítások varázslójából eltávolítható.
  • Szelektor felülírás gyorsítótár (chrome.storage.local): Szelektor hot-fix adatok, amelyeket a Beállításokban konfigurált URL-ről szereztek be (alapértelmezett: a Tutti GitHub Pages-en lévő nyilvános selectors.json).
  • Videofeltöltési limit gyorsítótár (chrome.storage.local): API szondák eredménye fiókonkénti videofeltöltési korlátokhoz (pl. Bluesky). Csak numerikus értékek; nincsenek hitelesítő adatok.

Mindezek bármikor törölhetők a bővítmény eltávolításával vagy a Chrome bővítményeinek tárhelyének törlésével.

3. Használt engedélyek

EngedélyCél
storageA fenti helyi adatok (beállítások, piszkozatok, előzmények, kiválasztott platformok stb.) tárolása a böngészőjén belül.
offscreenffmpeg.wasm futtatása offscreen dokumentumban videofájlok tömörítéséhez / újrakódolásához, amikor a forrás meghaladja a cél SNS méretkorlátját. A videoadatok csak helyileg kerülnek feldolgozásra; a Tutti sehova nem továbbítja őket.
sidePanelVálasztható UI megjelenítési mód: a Tutti bejegyzéskészítő űrlapját a Chrome oldalpaneljében jeleníti meg az akció popup helyett, így nyitva marad, miközben SNS lapok között vált. Az Opciók oldalon választhatja ki; az alapértelmezett a normál popup.
host_permissionsContent scriptek beillesztése minden SNS oldalra (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) a bejegyzéskészítő űrlap kitöltéséhez a bejegyzés tartalmával és az Ön nevében történő elküldéséhez. Nem olvasunk adatokat ezekről az oldalakról, és nem küldjük máshová; a content script csak az Ön saját bemenetét írja.
optional_host_permissions: https://*/*Csak akkor kerül lekérésre futásidőben, ha egyedi Mastodon vagy Misskey példány URL-t konfigurál (az alapértelmezetteken kívül). A Chrome ekkor megjeleníti a standard engedélyezési kérdést.

4. Kapcsolat az SNS szolgáltatókkal

A Tutti automatizálja a 11 támogatott hálózat webes UI-ját (vagy opcionálisan a hivatalos API-jukon keresztül posztol). Ez nem hivatalos funkcionalitás; a Tutti nem áll kapcsolatban az X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok vagy YouTube szolgáltatókkal. Továbbra is köteles az egyes szolgáltatások saját felhasználási feltételeinek betartására.
