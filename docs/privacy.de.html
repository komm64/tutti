Tutti — Datenschutzerklärung

Keine Drittanbieter-Datenerfassung. Alle Beitragsinhalte gehen direkt von Ihrem Browser an jedes SNS.

Startseite GitHub Support

Zuletzt aktualisiert: 2026-05-23 (v0.5.0)

1. Erhobene und übermittelte Daten

Tutti hat keinen Backend-Server. Beitragsinhalte (Text, Bilder, Video) werden niemals an einen Drittserver übermittelt. Sie wandern direkt von Ihrem Browser zu jedem ausgewählten sozialen Netzwerk, genau so, als hätten Sie sie dort selbst eingegeben und gesendet.

  • Beitragstext, Bilder und Video werden ausschließlich an die von Ihnen gewählten SNS-Seiten (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) innerhalb Ihres Browsers weitergegeben.
  • Wenn der optionale offizielle API-Pfad für Bluesky / Mastodon / Misskey aktiviert ist, werden Beiträge unter Verwendung der in den Einstellungen hinterlegten Zugangsdaten direkt aus Ihrem Browser an den eigenen API-Server dieses SNS gesendet. Kein Tutti- / Entwickler-Server ist beteiligt.
  • Es werden keine Analyse-, Tracking- oder Werbedaten erfasst.
  • Tutti liest oder speichert keine Authentifizierungstoken, Cookies oder Sitzungsdaten von den SNS-Seiten.

Ausnahme — manuelle Fehlerberichte (Zustimmung pro Klick)

Wenn (und nur wenn) Sie auf den Schaltfläche „Melden“ in einem Fehlerdialog oder im Diagnosepanel klicken, wird Folgendes über ein Cloudflare Workers-Relay an einen privaten GitHub-Issue-Tracker (komm64/tutti-issues, nur für den Tutti-Entwickler einsehbar) gesendet:

  • Die Fehlermeldung, die Erweiterungsversion, der User-Agent-String des Browsers.
  • Die letzten 30 Einträge des internen Erweiterungslogs (das von Ihnen konfigurierte Log-Level; standardmäßig INFO). Beitragsinhalte, Bilder, Video, SNS-Anmeldedaten und API-Zugangsdaten werden niemals einbezogen.
  • Ein redigierter DOM-Snapshot des SNS-Tabs, in dem der Fehler aufgetreten ist, zur Reparatur defekter Selektoren. Textinhalte, Attributwerte, URL-Pfade/-Abfragen/-Fragmente, href/src-Attribute und <meta>/<link>-Tags werden entfernt. Die Nutzlast ist auf 8 KB begrenzt. Siehe dom-snapshot.ts für die Implementierung.

Nichts wird gesendet, bevor Sie die Melden-Schaltfläche drücken. Das Ziel ist privat, nur für den Tutti-Entwickler einsehbar.

2. Lokal gespeicherte Daten

Folgendes wird ausschließlich innerhalb Ihres Browsers über die storage-API von Chrome gespeichert. Tutti übermittelt nichts vom Gerät weg.

  • Einstellungen (chrome.storage.sync): Mastodon-/Misskey-Instanz-URLs, autoPost-Schalter, Anzeigemodus, Selector-Override-URL usw. Kann über Chrome Sync auf Ihre anderen Geräte repliziert werden, wenn Sie in Chrome angemeldet sind.
  • Entwürfe (chrome.storage.session): In Bearbeitung befindlicher Beitragstext und angehängte Medien (Bilder / Video). Wird beim Beenden des Browsers automatisch gelöscht.
  • Ausgewählte SNS (chrome.storage.local): Die als Veröffentlichungsziele angehakten SNS. Bleibt der Bequemlichkeit halber sitzungsübergreifend bestehen; nur gerätelokal.
  • Zuletzt gesehene Benutzernamen (chrome.storage.local): Der aktuell angemeldete Benutzername bei jedem SNS, im Popup angezeigt, damit Sie bestätigen können, welches Konto den Beitrag erhalten wird. Nur gerätelokal.
  • Beitragsverlauf (chrome.storage.local): Die letzten 20 Beiträge: erste 80 Zeichen des Textes, Zielplattformen, Erfolg/Misserfolg pro Plattform und Zeitstempel. Nur gerätelokal.
  • API-Zugangsdaten (chrome.storage.local): Nur wenn Sie den offiziellen API-Pfad für Bluesky / Mastodon / Misskey ausdrücklich aktiviert haben: Ihr Bluesky App-Passwort oder Mastodon-/Misskey-Access-Token. Nur gerätelokal; niemals über Chrome Sync repliziert; niemals an einen Entwicklerserver übermittelt. Über den Einstellungsassistenten entfernbar.
  • Selector-Override-Cache (chrome.storage.local): Selector-Hotfix-Daten, die von der in den Einstellungen konfigurierten URL abgerufen werden (Standard: die öffentliche selectors.json auf Tuttis GitHub Pages).
  • Video-Upload-Limit-Cache (chrome.storage.local): Ergebnis von API-Abfragen für Video-Upload-Obergrenzen pro Konto (z. B. Bluesky). Nur numerische Werte; keine Zugangsdaten.

All das kann jederzeit gelöscht werden, indem Sie die Erweiterung deinstallieren oder den Speicher der Chrome-Erweiterungen leeren.

3. Verwendete Berechtigungen

BerechtigungZweck
storageDie oben genannten lokalen Daten (Einstellungen, Entwürfe, Verlauf, ausgewählte Plattformen usw.) innerhalb Ihres Browsers speichern.
offscreenffmpeg.wasm in einem Offscreen-Dokument ausführen, um Videodateien zu komprimieren / neu zu kodieren, wenn die Quelle die Größenbeschränkung des Ziel-SNS überschreitet. Videodaten werden ausschließlich lokal verarbeitet; werden von Tutti nirgendwohin übertragen.
sidePanelOptionaler UI-Anzeigemodus: Zeigt Tuttis Formular zur Beitragserstellung in der Seitenleiste von Chrome statt im Aktions-Popup an, damit es geöffnet bleibt, während Sie zwischen SNS-Tabs wechseln. Sie aktivieren dies über die Optionsseite; Standard ist das normale Popup.
host_permissionsContent Scripts in jede SNS-Site (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) injizieren, um das Formular zur Beitragserstellung mit Ihrem Beitragsinhalt zu füllen und in Ihrem Namen abzusenden. Es werden keine Daten von diesen Sites gelesen und woandershin gesendet; das Content Script schreibt nur Ihre eigene Eingabe.
optional_host_permissions: https://*/*Wird zur Laufzeit nur angefordert, wenn Sie eine benutzerdefinierte Mastodon- oder Misskey-Instanz-URL (außer den Standardwerten) konfigurieren. Chrome zeigt zu diesem Zeitpunkt die Standard-Berechtigungsabfrage an.

4. Beziehung zu den SNS-Anbietern

Tutti automatisiert die Web-UI (bzw. veröffentlicht optional über die offizielle API) der 11 unterstützten Netzwerke. Dies ist eine inoffizielle Funktionalität; Tutti ist weder mit X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok noch YouTube verbunden. Sie bleiben an die jeweiligen Nutzungsbedingungen der einzelnen Dienste gebunden.
