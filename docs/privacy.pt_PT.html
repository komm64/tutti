Tutti — Política de privacidade

Sem recolha de dados por terceiros. Todo o conteúdo das publicações vai diretamente do seu navegador para cada SNS.

Início GitHub Apoio

Última atualização: 2026-05-23 (v0.5.0)

1. Dados que recolhemos e transmitimos

O Tutti não tem servidor backend. O conteúdo das publicações (texto, imagens, vídeo) nunca é transmitido a qualquer servidor de terceiros. Viaja diretamente do seu navegador para cada rede social que selecionar, exatamente como se o tivesse escrito e submetido aí.

  • O texto, as imagens e o vídeo das publicações são passados apenas para as páginas dos SNS que escolher (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) dentro do seu navegador.
  • Quando o caminho opcional da API oficial está ativado para Bluesky / Mastodon / Misskey, as publicações são enviadas diretamente do seu navegador para o próprio servidor de API daquele SNS utilizando as credenciais fornecidas nas Definições. Nenhum servidor do Tutti / programador está envolvido.
  • Não são recolhidos dados analíticos, de seguimento ou publicitários.
  • O Tutti não lê nem armazena os seus tokens de autenticação, cookies ou dados de sessão dos sites SNS.

Exceção — relatórios manuais de erro (consentimento por clique)

Quando (e apenas quando) carrega no botão “Relatar” apresentado numa caixa de diálogo de erro ou no painel de diagnóstico, é enviado o seguinte através de um relay do Cloudflare Workers para um repositório privado de issues do GitHub (komm64/tutti-issues, visível apenas para o programador do Tutti):

  • A mensagem de erro, a versão da extensão, a cadeia User-Agent do navegador.
  • As últimas 30 entradas do registo interno da extensão (o nível de log configurado; INFO por omissão). O conteúdo das publicações, as imagens, o vídeo, os dados de início de sessão em SNS e as credenciais de API nunca são incluídos.
  • Uma captura DOM redigida do separador SNS onde ocorreu o erro, utilizada para reparar seletores partidos. O conteúdo de texto, os valores de atributos, os caminhos/consultas/fragmentos de URL, os atributos href/src e as etiquetas <meta>/<link> são removidos. O payload tem o limite de 8 KB. Consulte dom-snapshot.ts para a implementação.

Nada é enviado até que carregue no botão de relato. O destino é privado, visível apenas para o programador do Tutti.

2. Dados armazenados localmente

O seguinte é armazenado apenas dentro do seu navegador através da API storage do Chrome. Nada é transmitido para fora do dispositivo pelo Tutti.

  • Definições (chrome.storage.sync): URLs de instâncias Mastodon / Misskey, toggle autoPost, modo de visualização, URL de override de seletores, etc. Pode ser replicado para os seus outros dispositivos via Chrome Sync se tiver sessão iniciada no Chrome.
  • Rascunhos (chrome.storage.session): Texto de publicação em curso e ficheiros multimédia anexados (imagens / vídeo). Limpo automaticamente quando o navegador é fechado.
  • SNS selecionados (chrome.storage.local): Os SNS que marcou como destinos de publicação. Persiste entre sessões por conveniência; apenas local ao dispositivo.
  • Últimos nomes de utilizador vistos (chrome.storage.local): O nome de utilizador atualmente autenticado em cada SNS, apresentado no popup para que possa confirmar que conta receberá a publicação. Apenas local ao dispositivo.
  • Histórico de publicações (chrome.storage.local): As últimas 20 publicações: primeiros 80 caracteres de texto, plataformas-alvo, sucesso/falha por plataforma e timestamp. Apenas local ao dispositivo.
  • Credenciais de API (chrome.storage.local): Apenas se ativou explicitamente o caminho da API oficial para Bluesky / Mastodon / Misskey: a sua app password do Bluesky ou access token do Mastodon / Misskey. Apenas local ao dispositivo; nunca replicado via Chrome Sync; nunca transmitido a qualquer servidor do programador. Removível pelo assistente de Definições.
  • Cache de override de seletores (chrome.storage.local): Dados de hot-fix de seletores obtidos do URL que configurou nas Definições (omissão: o selectors.json público nas GitHub Pages do Tutti).
  • Cache do limite de envio de vídeo (chrome.storage.local): Resultado de sondagens API para limites de envio de vídeo por conta (por exemplo, Bluesky). Apenas valores numéricos; sem credenciais.

Tudo isto pode ser apagado a qualquer momento desinstalando a extensão ou limpando o armazenamento de extensões do Chrome.

3. Permissões utilizadas

PermissãoFinalidade
storageArmazenar os dados locais acima (definições, rascunhos, histórico, plataformas selecionadas, etc.) dentro do seu navegador.
offscreenExecutar ffmpeg.wasm num documento offscreen para comprimir / recodificar ficheiros de vídeo quando a origem exceder o limite de tamanho do SNS de destino. Os dados de vídeo são processados apenas localmente; não são transmitidos para lugar nenhum pelo Tutti.
sidePanelModo de visualização de UI opcional: apresenta o formulário de composição do Tutti no painel lateral do Chrome em vez do popup de ação, para que permaneça aberto enquanto alterna entre separadores SNS. Opta-se por ele a partir da página de Opções; a omissão é o popup padrão.
host_permissionsInjetar content scripts em cada site SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) para preencher o formulário de composição com o conteúdo da sua publicação e submetê-lo em seu nome. Não são lidos dados destes sites para enviar para outro lado; o content script apenas escreve a sua própria entrada.
optional_host_permissions: https://*/*Pedido em tempo de execução apenas se configurar um URL de instância personalizado de Mastodon ou Misskey (diferente das omissões). O Chrome mostra o pedido de permissão padrão nesse momento.

4. Relação com os fornecedores SNS

O Tutti automatiza a UI web das (ou, opcionalmente, publica via API oficial das) 11 redes suportadas. Esta é funcionalidade não oficial; o Tutti não está associado a X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok ou YouTube. Permanece sujeito aos termos de utilização de cada serviço.
