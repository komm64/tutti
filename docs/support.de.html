Brauchen Sie Hilfe?

1. Veröffentlichung fehlgeschlagen / etwas verhält sich seltsam

Verwenden Sie die Schaltfläche „Diagnose“ → „Melden“ im Popup. Dies ist der kanonische Kanal. Der Bericht wird redigiert (Handles, URLs, E-Mails) und an unseren privaten Issue-Tracker weitergeleitet, wo automatisch ein PR zur Selektor-Korrektur erstellt und innerhalb weniger Tage als Hotfix an alle Nutzer ausgeliefert wird.

Warum die Melden-Schaltfläche im Produkt verwenden?

Der Melden-Ablauf des Popups hängt automatisch den endgültigen Erstellungszustand, einen redigierten DOM-Snapshot und die Selektor-Audit-Ergebnisse an — genau das, was zur schnellen Behebung von Selektorausfällen benötigt wird. PII wird in mehreren Schichten vor dem Senden entfernt.

2. Allgemeine Fragen / Funktionswünsche

Schreiben Sie uns an contact@komm64.com. Dies ist ein privater Kanal, sicher, um Screenshots / persönliche Handles zu teilen.

3. Fragen zur Installation / zum Konto

Tutti nutzt Ihre bestehenden Browser-Anmeldungen wieder für jedes soziale Netzwerk. Es gibt kein separates Tutti-Konto und keinen Anmeldeschritt. Sobald Sie in demselben Chrome-Profil bei einem sozialen Netzwerk angemeldet sind, kann Tutti von dort aus veröffentlichen.

FAQ

F. Meine Veröffentlichung ist fehlgeschlagen.

Das bedeutet normalerweise, dass ein soziales Netzwerk sein DOM geändert hat. Verwenden Sie Diagnose → Melden im Popup. Ein Selektor-Hotfix wird in der Regel innerhalb weniger Tage über das automatische Update von selectors.json ausgeliefert (kein Erweiterungs-Update erforderlich).

F. Tutti merkt sich meine Anmeldung nicht.

Tutti stützt sich auf die normale Browser-Anmeldung des Netzwerks. Melden Sie sich zuerst in Chrome bei jedem Netzwerk an, dann kann Tutti in Ihrem Namen veröffentlichen. Der Inkognito-Modus funktioniert nicht, da die Sitzungen des Netzwerks dort nicht vorhanden sind.

F. Die Zeichenbegrenzungen unterscheiden sich je nach Netzwerk — ist das ein Problem?

Nein. Lange Beiträge, die die Zeichenbegrenzung eines Netzwerks überschreiten, werden automatisch in einen Fortsetzungs-Thread aufgeteilt (bei X eine richtige Antwortkette; bei anderen Netzwerken sequentielle Beiträge). Beiträge innerhalb des Limits werden so wie sie sind durchgereicht.

F. Mir wird gesagt, dass mein Bild / Video zu groß ist.

Bilder werden über Canvas automatisch an das Größenlimit jedes Netzwerks angepasst. Videos werden mit ffmpeg.wasm automatisch komprimiert, wenn sie über dem Limit eines Netzwerks liegen. Wenn selbst die Komprimierung nicht passt, sehen Sie einen Fehler — bitte überprüfen Sie zuerst, ob die Datei über die native App des Netzwerks hochgeladen werden kann.

F. Sieht Tutti meinen Beitragsinhalt?

Nein. Beitragsinhalte laufen nie über einen Drittserver. Tutti injiziert direkt in die Web-Erstellungsseite jedes Netzwerks. Siehe unsere Datenschutzerklärung für Details. Die einzige Ausnahme ist, wenn Sie die im Produkt integrierte Melden-Schaltfläche drücken, die redigierte Fehlerinformationen an unseren privaten Issue-Tracker sendet.

F. Kann ich spenden?

Noch nicht — wir arbeiten an einem unaufdringlichen Support-Kanal. Die größte laufende Kosten ist es, mit den UI-Änderungen jedes sozialen Netzwerks Schritt zu halten; künftige Ko-fi-Abläufe werden diese Wartung finanzieren.
