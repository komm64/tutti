困ったときは

1. 投稿でエラーが出た / 期待通り動かない

popup の 「診断」 ボタン → 「報告する」 から送ってください。 これが正規経路です。 内部で個人ハンドル / URL / メールアドレスを redact し、 私たちの private な issue tracker に届きます。 そこから自動で selector 修正の PR が作られて、 数日中に hot-fix が全ユーザーに配信される仕組みになっています。

なぜ正規経路を使ってほしい?

popup の Report ボタンを経由した報告には、 投稿の最終状態 / DOM snapshot / selector 検出結果が自動添付されます。 これがあると selector breakage の修正が 桁違いに早くなります。 個人特定可能な情報は送信前に多段で redact されます。

2. 使い方の質問 / 機能の要望

contact@komm64.com に直接メールしてください。 個人 SNS のスクショ / ハンドル等を貼っても外部に漏れない経路です。

3. インストール / アカウント関連

Tutti は 各 SNS の既存ブラウザログインを再利用します。 Tutti 側でアカウントを登録する手順はありません。 popup を開いて投稿先 SNS にチェックを入れたタイミングで、 該当 SNS にすでにログイン済の Chrome 拡張 セッションがそのまま使われます。

よくある質問

Q. 投稿が失敗しました

SNS 側の DOM 構造が変わった可能性があります。 popup の診断ボタン → 報告する で送ってください。 多くの場合、 数日中に hot-fix が selectors.json 経由で自動配信されます (拡張本体の更新は不要)。

Q. ログインを覚えてくれない

Tutti は各 SNS の Chrome ログイン状態をそのまま使います。 SNS タブで先に ブラウザログインしてから Tutti を使ってください。 シークレットモードでは SNS 側のログインが切れているので動きません。

Q. 投稿の文字数が SNS によって違うけど大丈夫?

各 SNS の文字数上限を超えた長文は自動でスレッド分割されます (X は reply chain、 他 SNS は連投)。 上限以内ならそのまま投稿されます。

Q. 画像 / 動画のサイズが大きすぎると言われる

画像は各 SNS のサイズ制限に合わせて Canvas で自動リサイズされます。 動画は SNS 別の上限を超える場合、 ffmpeg.wasm で自動圧縮を試みます。 それでも 上限を超える動画はエラーになります。 SNS のネイティブアプリで一度 アップロードできるサイズか確認してください。

Q. Tutti は私の投稿内容を見ていますか?

いいえ。 投稿内容は第三者サーバーを一切経由しません。 各 SNS の Web 投稿 ページに直接 inject されます。 詳細はプライバシー ポリシーを参照してください。 唯一例外は popup の「報告する」 ボタンを 押した時のみで、 これは redact 済のエラー情報を private な issue tracker に 転送します。

Q. 寄付 / 支援 はできますか?

現在準備中です。 まずは Tutti を使い続けてもらえれば嬉しいです。 SNS UI の変更に追従しつづける継続保守がコストの中心なので、 安定した 品質を維持するために将来 Ko-fi 等の支援導線を設ける予定です。
