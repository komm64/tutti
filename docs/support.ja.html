困ったときは

1. 投稿が失敗した / 動作が変

popup の「診断」 → 「報告する」ボタンをお使いください。これが正規経路です。レポートは redact (ハンドル・URL・メール) 済みで private な issue tracker に転送され、そこから自動的に selector 修正 PR が作成され、数日中に hot-fix として全ユーザーに配信されます。

なぜ製品内の Report ボタンを?

popup の Report 経路は投稿の最終状態、redacted DOM snapshot、selector 検出結果を自動添付します。これが selector breakage の修正に必要なすべてです。個人特定可能な情報は送信前に多段で redact されます。

2. 一般的な質問 / 機能要望

contact@komm64.com までメールしてください。private な経路です。スクリーンショットや個人ハンドルを安全に共有できます。

3. インストール / アカウント関連

Tutti は 各 SNS の既存ブラウザログインを再利用します。Tutti 専用のアカウントやログイン手順はありません。同じ Chrome プロファイルで SNS にログイン済みなら、そこから Tutti が投稿できます。

FAQ

Q. 投稿が失敗しました。

多くの場合、SNS が DOM 構造を変更したためです。popup の 診断 → 報告する をお使いください。多くの場合 selectors.json 自動更新で数日中に hot-fix が配信されます (拡張本体の更新は不要)。

Q. Tutti がログインを覚えてくれない。

Tutti は各 SNS の通常のブラウザログインに依存します。まず各 SNS に Chrome でログインしてください。そうすれば Tutti が代わりに投稿できます。シークレットモードでは SNS のセッションが存在しないため動きません。

Q. SNS によって文字数制限が違うけど問題は?

問題ありません。文字数制限を超える長文は自動で連投スレッドに分割されます (X は真の reply chain、他 SNS は連投)。制限以内ならそのまま投稿されます。

Q. 画像 / 動画が大きすぎると言われる。

画像は Canvas で各 SNS のサイズ上限に合わせて自動リサイズされます。動画は SNS の上限を超える場合 ffmpeg.wasm で自動圧縮されます。それでも収まらない場合はエラーになります。SNS のネイティブアプリでアップロードできるサイズか先に確認してください。

Q. Tutti は投稿内容を見ていますか?

いいえ。投稿内容は第三者サーバーを一切経由しません。Tutti は各 SNS の Web 投稿ページに直接 inject します。詳細はプライバシーポリシーを参照。唯一の例外は製品内の Report ボタンを押した時で、これは redact 済のエラー情報を private な issue tracker に送信します。

Q. 寄付できますか?

まだです — プレッシャーの少ないサポート経路を準備中です。最大の継続コストは各 SNS の UI 変更への追従なので、将来の Ko-fi 等はそのメンテナンス費用に充てられます。
