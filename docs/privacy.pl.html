Tutti — Polityka prywatności

Brak gromadzenia danych przez strony trzecie. Cała zawartość postów trafia bezpośrednio z Twojej przeglądarki do każdej SNS.

Strona główna GitHub Pomoc

Ostatnia aktualizacja: 2026-05-23 (v0.5.0)

1. Dane, które zbieramy i przekazujemy

Tutti nie ma serwera backend. Zawartość postów (tekst, obrazy, wideo) nigdy nie jest przesyłana do żadnego serwera strony trzeciej. Podróżuje bezpośrednio z Twojej przeglądarki do każdej wybranej sieci społecznościowej, dokładnie tak, jakbyś sam(a) wpisał(a) i wysłał(a) to tam.

  • Tekst, obrazy i wideo postów są przekazywane tylko do wybranych przez Ciebie stron SNS (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) wewnątrz Twojej przeglądarki.
  • Gdy opcjonalna ścieżka oficjalnego API jest włączona dla Bluesky / Mastodon / Misskey, posty są wysyłane bezpośrednio z Twojej przeglądarki na własny serwer API tego SNS przy użyciu poświadczeń, które podałeś w Ustawieniach. Żaden serwer Tutti / dewelopera nie jest zaangażowany.
  • Nie są gromadzone żadne dane analityczne, śledzenia ani reklamowe.
  • Tutti nie odczytuje ani nie przechowuje Twoich tokenów uwierzytelniania, plików cookie ani danych sesji z witryn SNS.

Wyjątek — ręczne raporty o błędach (zgoda przy każdym kliknięciu)

Gdy (i tylko gdy) naciśniesz przycisk „Zgłoś” pokazany w oknie błędu lub panelu diagnostyki, następujące dane są wysyłane przez przekaźnik Cloudflare Workers do prywatnego trackera zgłoszeń GitHub (komm64/tutti-issues, widocznego tylko dla dewelopera Tutti):

  • Komunikat o błędzie, wersja rozszerzenia, ciąg User-Agent przeglądarki.
  • Ostatnie 30 wpisów wewnętrznego logu rozszerzenia (poziom logu, który skonfigurowałeś; domyślnie INFO). Zawartość postów, obrazy, wideo, dane logowania do SNS i poświadczenia API nigdy nie są dołączane.
  • Zredagowany snapshot DOM karty SNS, w której wystąpił błąd, używany do naprawy zepsutych selektorów. Zawartość tekstowa, wartości atrybutów, ścieżki/zapytania/fragmenty URL, atrybuty href/src oraz tagi <meta>/<link> są usuwane. Ładowność jest ograniczona do 8 KB. Implementację zobacz w dom-snapshot.ts.

Nic nie jest wysyłane, dopóki nie naciśniesz przycisku zgłoszenia. Adresat jest prywatny, widoczny tylko dla dewelopera Tutti.

2. Dane przechowywane lokalnie

Poniższe dane są przechowywane wyłącznie wewnątrz Twojej przeglądarki za pośrednictwem API storage Chrome. Nic nie jest przesyłane poza urządzenie przez Tutti.

  • Ustawienia (chrome.storage.sync): URL-e instancji Mastodon / Misskey, przełącznik autoPost, tryb wyświetlania, URL nadpisania selektorów itd. Mogą być replikowane na Twoje inne urządzenia przez Chrome Sync, jeśli jesteś zalogowany do Chrome.
  • Wersje robocze (chrome.storage.session): Tekst postu w trakcie i załączone media (obrazy / wideo). Automatycznie czyszczone po zamknięciu przeglądarki.
  • Wybrane SNS (chrome.storage.local): SNS, które zaznaczyłeś jako cele publikacji. Utrzymuje się między sesjami dla wygody; tylko lokalnie na urządzeniu.
  • Ostatnio widziane nazwy użytkowników (chrome.storage.local): Aktualnie zalogowana nazwa użytkownika w każdym SNS, wyświetlana w popup, abyś mógł potwierdzić, które konto otrzyma post. Tylko lokalnie na urządzeniu.
  • Historia postów (chrome.storage.local): Ostatnie 20 postów: pierwsze 80 znaków tekstu, platformy docelowe, sukces/niepowodzenie na platformę i znacznik czasu. Tylko lokalnie na urządzeniu.
  • Poświadczenia API (chrome.storage.local): Tylko jeśli jawnie włączyłeś ścieżkę oficjalnego API dla Bluesky / Mastodon / Misskey: Twoje Bluesky app password lub token dostępu Mastodon / Misskey. Tylko lokalnie na urządzeniu; nigdy nie replikowane przez Chrome Sync; nigdy nie przesyłane do żadnego serwera dewelopera. Można usunąć z kreatora Ustawień.
  • Cache nadpisania selektorów (chrome.storage.local): Dane hot-fixu selektorów pobierane z URL skonfigurowanego w Ustawieniach (domyślnie: publiczny selectors.json na GitHub Pages Tutti).
  • Cache limitu przesyłania wideo (chrome.storage.local): Wynik sond API dla limitów przesyłania wideo na konto (np. Bluesky). Tylko wartości liczbowe; brak poświadczeń.

Wszystko to można wyczyścić w dowolnym momencie, odinstalowując rozszerzenie lub czyszcząc pamięć rozszerzeń Chrome.

3. Używane uprawnienia

UprawnienieCel
storagePrzechowywanie powyższych danych lokalnych (ustawienia, wersje robocze, historia, wybrane platformy itd.) wewnątrz Twojej przeglądarki.
offscreenUruchamianie ffmpeg.wasm w dokumencie offscreen do kompresji / przekodowania plików wideo, gdy źródło przekracza limit rozmiaru docelowego SNS. Dane wideo są przetwarzane tylko lokalnie; nie są przesyłane nigdzie przez Tutti.
sidePanelOpcjonalny tryb wyświetlania UI: pokazuje formularz tworzenia postu Tutti w panelu bocznym Chrome zamiast popup akcji, aby pozostawał otwarty podczas przełączania między kartami SNS. Włączasz go na stronie Opcji; domyślnie jest to standardowy popup.
host_permissionsWstrzykiwanie content scripts do każdej witryny SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) w celu wypełnienia formularza tworzenia postu zawartością Twojego postu i wysłania go w Twoim imieniu. Z tych witryn nie są odczytywane żadne dane i wysyłane gdzie indziej; content script zapisuje tylko Twoje własne dane wejściowe.
optional_host_permissions: https://*/*Wymagane w czasie wykonywania tylko jeśli skonfigurujesz niestandardowy URL instancji Mastodon lub Misskey (inny niż domyślny). Chrome wyświetla standardowy monit o uprawnienie w tym momencie.

4. Relacja z dostawcami SNS

Tutti automatyzuje interfejs webowy (lub, opcjonalnie, publikuje przez oficjalne API) 11 obsługiwanych sieci. To nieoficjalna funkcjonalność; Tutti nie jest powiązany z X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok ani YouTube. Nadal obowiązują Cię własne warunki użytkowania każdej usługi.
