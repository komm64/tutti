Tutti — Politica de confidențialitate

Nicio colectare de date de către terți. Tot conținutul postărilor merge direct din browserul tău către fiecare SNS.

Acasă GitHub Asistență

Ultima actualizare: 2026-05-23 (v0.5.0)

1. Datele pe care le colectăm și le transmitem

Tutti nu are server backend. Conținutul postărilor (text, imagini, video) nu este niciodată transmis către niciun server terț. Călătorește direct din browserul tău către fiecare rețea socială pe care o selectezi, exact ca și cum l-ai fi tastat și trimis tu însuți acolo.

  • Textul, imaginile și videoul postării sunt transmise doar paginilor SNS pe care le alegi (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) în interiorul browserului tău.
  • Când calea API oficială opțională este activată pentru Bluesky / Mastodon / Misskey, postările sunt trimise direct din browserul tău către propriul server API al acelei SNS folosind credențialele furnizate în Setări. Niciun server Tutti / al dezvoltatorului nu este implicat.
  • Nu sunt colectate date de analiză, urmărire sau publicitate.
  • Tutti nu citește și nu stochează tokenurile tale de autentificare, cookie-urile sau datele de sesiune de pe site-urile SNS.

Excepție — rapoarte manuale de erori (consimțământ per clic)

Când (și doar când) apeși butonul „Raportează” afișat într-un dialog de eroare sau în panoul de diagnostic, următoarele sunt trimise printr-un relay Cloudflare Workers către un tracker de issue-uri GitHub privat (komm64/tutti-issues, vizibil doar pentru dezvoltatorul Tutti):

  • Mesajul de eroare, versiunea extensiei, șirul User-Agent al browserului.
  • Ultimele 30 de intrări din jurnalul intern al extensiei (nivelul de log pe care l-ai configurat; INFO implicit). Conținutul postărilor, imaginile, videoul, datele de autentificare SNS și credențialele API niciodată nu sunt incluse.
  • Un snapshot DOM editat al tab-ului SNS în care a apărut eroarea, folosit pentru repararea selectorilor stricați. Conținutul textului, valorile atributelor, căile/interogările/fragmentele URL, atributele href/src și tag-urile <meta>/<link> sunt eliminate. Payload-ul este limitat la 8 KB. Vezi dom-snapshot.ts pentru implementare.

Nimic nu se trimite până nu apeși butonul de raportare. Destinația este privată, vizibilă doar pentru dezvoltatorul Tutti.

2. Date stocate local

Următoarele sunt stocate doar în browserul tău prin API-ul storage al Chrome. Nimic nu este transmis în afara dispozitivului de către Tutti.

  • Setări (chrome.storage.sync): URL-uri ale instanțelor Mastodon / Misskey, comutator autoPost, mod de afișare, URL de suprascriere selectori etc. Pot fi replicate pe celelalte dispozitive prin Chrome Sync dacă ești conectat la Chrome.
  • Ciorne (chrome.storage.session): Text postare în desfășurare și media atașate (imagini / video). Șterse automat la închiderea browserului.
  • SNS selectate (chrome.storage.local): SNS-urile pe care le-ai bifat ca ținte de postare. Persistă între sesiuni pentru comoditate; doar local pe dispozitiv.
  • Ultimele nume de utilizator văzute (chrome.storage.local): Numele de utilizator conectat curent pe fiecare SNS, afișat în popup pentru a putea confirma ce cont va primi postarea. Doar local pe dispozitiv.
  • Istoric postări (chrome.storage.local): Ultimele 20 de postări: primele 80 de caractere ale textului, platforme țintă, succes/eșec per platformă și marcaj temporal. Doar local pe dispozitiv.
  • Credențiale API (chrome.storage.local): Doar dacă ai activat explicit calea API oficială pentru Bluesky / Mastodon / Misskey: Bluesky app password sau access token Mastodon / Misskey. Doar local pe dispozitiv; niciodată replicate prin Chrome Sync; niciodată transmise vreunui server al dezvoltatorului. Pot fi eliminate din asistentul de Setări.
  • Cache suprascriere selectori (chrome.storage.local): Date hot-fix pentru selectori obținute de la URL-ul configurat în Setări (implicit: selectors.json public pe GitHub Pages al Tutti).
  • Cache limită de încărcare video (chrome.storage.local): Rezultatul sondelor API pentru limitele de încărcare video per cont (de ex. Bluesky). Doar valori numerice; nicio credențială.

Toate acestea pot fi șterse oricând prin dezinstalarea extensiei sau curățarea spațiului de stocare al extensiilor Chrome.

3. Permisiuni folosite

PermisiuneScop
storageStocarea datelor locale de mai sus (setări, ciorne, istoric, platforme selectate etc.) în browserul tău.
offscreenRularea ffmpeg.wasm într-un document offscreen pentru comprimarea / recodificarea fișierelor video atunci când sursa depășește limita de dimensiune a SNS-ului țintă. Datele video sunt procesate doar local; Tutti nu le transmite nicăieri.
sidePanelMod de afișare UI opțional: arată formularul de compunere al Tutti în panoul lateral Chrome în loc de popup-ul de acțiune, astfel încât să rămână deschis în timp ce comuți între tab-uri SNS. Activezi din pagina Opțiuni; valoarea implicită este popup-ul standard.
host_permissionsInjectarea de content scripts pe fiecare site SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) pentru a completa formularul de compunere cu conținutul postării tale și a-l trimite în numele tău. Nu sunt citite date de pe aceste site-uri și trimise în altă parte; content script scrie doar propria ta intrare.
optional_host_permissions: https://*/*Solicitat la rulare doar dacă configurezi un URL de instanță personalizat pentru Mastodon sau Misskey (altul decât cele implicite). Chrome afișează promptul standard de permisiune în acel moment.

4. Relația cu furnizorii SNS

Tutti automatizează interfața web (sau, opțional, postează prin API-ul oficial) al celor 11 rețele suportate. Aceasta este funcționalitate neoficială; Tutti nu este afiliat cu X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok sau YouTube. Rămâi legat de termenii de utilizare proprii ai fiecărui serviciu.
