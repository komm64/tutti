Tutti — Zásady ochrany soukromí

Žádný sběr dat třetími stranami. Veškerý obsah příspěvků jde přímo z vašeho prohlížeče do každé SNS.

Domů GitHub Podpora

Poslední aktualizace: 2026-05-23 (v0.5.0)

1. Údaje, které shromažďujeme a přenášíme

Tutti nemá backend server. Obsah příspěvků (text, obrázky, video) není nikdy přenášen na žádný server třetí strany. Putuje přímo z vašeho prohlížeče do každé vybrané sociální sítě, přesně jako byste je tam sami napsali a odeslali.

  • Text, obrázky a video příspěvků jsou předávány pouze na vámi zvolené stránky SNS (X, Bluesky, Threads, Mastodon, Misskey, Tumblr, Pixiv, DeviantArt, Instagram, TikTok, YouTube) uvnitř vašeho prohlížeče.
  • Když je volitelná cesta oficiálního API povolena pro Bluesky / Mastodon / Misskey, příspěvky jsou odesílány přímo z vašeho prohlížeče na vlastní API server této SNS pomocí přihlašovacích údajů, které jste poskytli v Nastavení. Žádný server Tutti / vývojáře není zapojen.
  • Nejsou shromažďována žádná analytická, sledovací ani reklamní data.
  • Tutti nečte ani neukládá vaše ověřovací tokeny, soubory cookie ani data relace ze stránek SNS.

Výjimka — ruční hlášení chyb (souhlas na klik)

Když (a pouze když) stisknete tlačítko „Nahlásit“ zobrazené v dialogu chyby nebo panelu diagnostiky, je následující odesláno přes relé Cloudflare Workers do privátního trackeru issue GitHub (komm64/tutti-issues, viditelného pouze vývojáři Tutti):

  • Chybová zpráva, verze rozšíření, řetězec User-Agent prohlížeče.
  • Posledních 30 záznamů z vnitřního logu rozšíření (úroveň logu, kterou jste nastavili; výchozí INFO). Obsah příspěvků, obrázky, video, přihlašovací data SNS a přihlašovací údaje API nikdy nejsou zahrnuty.
  • Redigovaný DOM snapshot karty SNS, kde došlo k chybě, používaný k opravě poškozených selektorů. Textový obsah, hodnoty atributů, cesty/dotazy/fragmenty URL, atributy href/src a tagy <meta>/<link> jsou odstraněny. Užitečné zatížení je omezeno na 8 KB. Implementaci viz dom-snapshot.ts.

Nic se neodesílá, dokud nestisknete tlačítko hlášení. Cílové místo je privátní, viditelné pouze vývojáři Tutti.

2. Lokálně ukládaná data

Následující se ukládá pouze uvnitř vašeho prohlížeče prostřednictvím API storage Chrome. Nic není přenášeno mimo zařízení Tutti.

  • Nastavení (chrome.storage.sync): URL instancí Mastodon / Misskey, přepínač autoPost, režim zobrazení, URL přepsání selektorů atd. Může být replikováno na vaše další zařízení přes Chrome Sync, pokud jste přihlášeni do Chrome.
  • Koncepty (chrome.storage.session): Text rozepsaného příspěvku a připojená média (obrázky / video). Automaticky se vymaže při ukončení prohlížeče.
  • Vybrané SNS (chrome.storage.local): SNS, které jste zaškrtli jako cíle publikování. Zachovává se mezi relacemi pro pohodlí; pouze lokálně na zařízení.
  • Posledně viděná uživatelská jména (chrome.storage.local): Aktuálně přihlášené uživatelské jméno na každé SNS, zobrazované v popup, abyste mohli potvrdit, který účet obdrží příspěvek. Pouze lokálně na zařízení.
  • Historie příspěvků (chrome.storage.local): Posledních 20 příspěvků: prvních 80 znaků textu, cílové platformy, úspěch/selhání na platformu a časové razítko. Pouze lokálně na zařízení.
  • Přihlašovací údaje API (chrome.storage.local): Pouze pokud jste explicitně povolili cestu oficiálního API pro Bluesky / Mastodon / Misskey: vaše Bluesky app password nebo přístupový token Mastodon / Misskey. Pouze lokálně na zařízení; nikdy nereplikováno přes Chrome Sync; nikdy nepřenášeno na žádný vývojářský server. Lze odebrat z průvodce Nastavení.
  • Cache přepsání selektorů (chrome.storage.local): Data hot-fixu selektorů získaná z URL, kterou jste nakonfigurovali v Nastavení (výchozí: veřejný selectors.json na GitHub Pages Tutti).
  • Cache limitu nahrávání videa (chrome.storage.local): Výsledek API sond pro limity nahrávání videa na účet (např. Bluesky). Pouze číselné hodnoty; žádné přihlašovací údaje.

Vše lze kdykoli vymazat odinstalováním rozšíření nebo vymazáním úložiště rozšíření Chrome.

3. Použitá oprávnění

OprávněníÚčel
storageUkládat výše uvedená lokální data (nastavení, koncepty, historie, vybrané platformy atd.) uvnitř vašeho prohlížeče.
offscreenSpouštět ffmpeg.wasm v offscreen dokumentu pro kompresi / překódování videosouborů, když zdroj překračuje limit velikosti cílové SNS. Videodata jsou zpracovávána pouze lokálně; Tutti je nikam nepřenáší.
sidePanelVolitelný režim zobrazení UI: zobrazuje formulář pro vytvoření příspěvku Tutti v bočním panelu Chrome místo akčního popup, takže zůstane otevřený při přepínání mezi kartami SNS. Aktivujete jej ze stránky Možnosti; výchozí je standardní popup.
host_permissionsVkládat content scripts na každý web SNS (x.com / twitter.com / bsky.app / bsky.social / threads.net / threads.com / mastodon.social / misskey.io / tumblr.com / pixiv.net / tiktok.com / youtube.com / instagram.com / deviantart.com) pro vyplnění formuláře pro vytvoření příspěvku obsahem vašeho příspěvku a jeho odeslání vaším jménem. Z těchto webů se nečtou žádná data ani se neodesílají jinam; content script pouze zapisuje váš vlastní vstup.
optional_host_permissions: https://*/*Požadováno za běhu pouze pokud nakonfigurujete vlastní URL instance Mastodon nebo Misskey (jinou než výchozí). Chrome v tu chvíli zobrazí standardní výzvu k udělení oprávnění.

4. Vztah s poskytovateli SNS

Tutti automatizuje webové UI (nebo volitelně publikuje přes oficiální API) 11 podporovaných sítí. Toto je neoficiální funkčnost; Tutti není přidružen k X, Bluesky, Meta (Threads / Instagram), Mastodon gGmbH, Misskey, Tumblr, Pixiv, DeviantArt, TikTok ani YouTube. Stále se vás týkají vlastní podmínky použití každé služby.
